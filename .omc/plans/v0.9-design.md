**File**: `tests/test_a10a_soundcam_corner.py` (NEW; default-lane).

Test count: **+3 default-lane tests**.

1. `test_a10a_soundcam_lab_corner_under_10cm` — for room `lab`,

   corner-vs-corner test below is the always-on path**). Run mesh

2. `test_a10a_soundcam_living_room_corner_under_10cm` — same.
3. `test_a10a_soundcam_conference_corner_under_10cm` — same.

**Default-lane behaviour**: each test has TWO modes:
- **Cached mode** (`ROOMESTIM_SOUNDCAM_DIR` unset): the GT JSON is
  authored *by running the algorithm against the mesh once at
  fixture-authoring time*; the test re-asserts that the JSON's
  corners are self-consistent (e.g., they form a non-degenerate
  polygon, distance between consecutive corners > 0.5 m, ceiling
  height > 1.5 m). This is a "fixture-integrity" assertion that runs
  every CI invocation without requiring a multi-GB download.
- **Live mode** (`ROOMESTIM_SOUNDCAM_DIR` set): re-runs the algorithm
  against the actual PLY mesh and asserts ≤ 10 cm vs the cached GT
  JSON. This is the substantive A10a numerical claim.

The `< 10 cm vs Azure-Kinect-mesh-derived GT` framing is recorded in
the test docstring + ADR 0016: the GT is the upstream textured mesh
itself (not a separate hand-tape measurement), so this is a substitute
for A10's hand-tape gate. The substitute caveat is preserved
explicitly.

### §2.5 ADR 0016 — Stage-2 schema flip via public-dataset A10a substitute

**File**: `docs/adr/0016-stage2-schema-flip-via-public-dataset-substitute.md` (NEW).

Required sections:
- Status (Accepted v0.9.0).
- Date (2026-05-09).
- Predecessor (v0.8.0 `5cd5be3`; ADR 0001..0015; D1..D19; OQ-4 / OQ-11).
- Decision: `__schema_version__` flips `"0.1-draft"` → `"0.1"`;
  Stage-2 schema (`proto/room_schema.json`, already Stage 2 strict)
  becomes canonical; the flip is **substitute-driven** (SoundCam 3
  rooms × A10a corner ≤ 10 cm) NOT in-situ-driven (A10b user lab
  unchanged).
- Drivers (3-5 bullets):
  1. v0.1.1 already authored Stage-2 schema FILE infrastructure
     (`additionalProperties: false`, `version: const "0.1"`); the only
     missing piece for 8 releases has been the marker flip.
  2. A10 has been blocked for 8 releases on the in-situ premise; v0.8
     strategic-position report identified this as the 0%-progress
     deadlock blocking spatial_engine integration + cross-repo PR.
  3. SoundCam (MIT, 3 rooms with mesh + measured RT60) provides
     license-clean substitute evidence at < 10 cm corner accuracy
     using the existing Polycam-adapter mesh→polygon path; this
     unblocks Stage-2 flip without requiring user lab access.
  4. Stage-2 flip is the prerequisite for the v0.2.0 cross-repo PR
     that has been frozen as draft for 7 releases; flipping
     unblocks the spatial_engine `proto/room_schema.json` proposal.
- Alternatives considered (≥4):
  - (a) Wait for A10b in-situ — rejected; 8-release deadlock pattern
    explicitly identified as drift in v0.8 strategic report; the
    project pivots from "wait for capture" to "use clean public
    substitute" as the v0.9 minimum-leverage move.
  - (b) Use ARKitScenes instead — rejected; license is Apple
    non-commercial (risks spatial_engine future commercial path);
    scope is 1,661 scenes × hundreds of GB (out of scope); the
    minimum-leverage move is 3 SoundCam rooms first.
  - (c) Use Motus instead — rejected; 1 room only with HOA RIR + OBJ
    CAD; phone-scan path completely uninvolved; adapter not
    exercised.
  - (d) Flip Stage-2 schema without ANY substitute (just declare
    "schema is good enough on author judgement") — rejected; would
    break the OQ-4 / D8 / D2 condition that Stage-2 lock requires
    real-world fixture exercise (≥3 captures per D2 reverse rule).
    SoundCam provides exactly that (≥3 rooms).
  - (e) Defer Stage-2 flip another release — rejected; the v0.7
    critic + v0.8 strategic report both flagged the "deferred list
    keeps growing" anti-pattern; the v0.9 minimum-leverage move is
    to actually flip when substitute evidence is sufficient.
- Why chosen — minimum-leverage answer to v0.8 strategic report;
  preserves the 8-release schema-file infrastructure investment;
  unblocks cross-repo PR; substitute-vs-in-situ honesty preserved
  via reverse criterion below.
- Consequences:
  - `__schema_version__` becomes `"0.1"`; `RoomModel.schema_version`
    default becomes `"0.1"`.
  - New writes emit `version: "0.1"`.
  - Old reads (`version: "0.1-draft"`) continue to validate against
    the draft schema (backward-compat retained via existing
    `_load_schema` switch at `roomestim/io/room_yaml_reader.py:32`).
  - `additionalProperties: false` is now in force on Stage-2 reads;
    consumers writing extensions must use Stage-2 explicit extension
    fields (e.g., `mount_surfaces` already-defined slot, or future
    `x_*` namespaced extensions added in a v0.9+ schema patch).

**File**: `docs/adr/0017-a10-layout-deferred-classification.md` (NEW).

- Decision: A10-layout (VBAP-N speakers ±5° azimuth + ±10 cm radial
  vs PHYSICAL speakers in a real room) is **explicitly classified
  as not substitutable by any public dataset**. v0.9 ships A10a
  (corner) and A11-boost (RT60); A10-layout remains DEFERRED to
  user in-situ work and is NOT advertised as part of A10a closure.
- Drivers (3-5 bullets):
  1. No public dataset known to v0.9 ships VBAP-N layout GT against

  2. Pretending A10a substitute closes A10-layout would dishonestly
     claim acceptance for a verification path that was not exercised.
  3. The strategic-position report scorecard (§3 row A10) explicitly
     decomposes A10 into geometry + layout sub-criteria; ADR 0017
     formalises this decomposition.
  4. Layout verification fundamentally requires PHYSICAL placement
     of speakers in a real captured room, not a public-dataset
     substitute (ground-truth speaker positions are not part of
     room-scan datasets).
- Alternatives considered (≥3):
  - (a) Synthesise speaker positions on SoundCam rooms (e.g., place
    8 virtual VBAP speakers + check the *coordinates* against
    geometric correctness) — rejected; this is unit-test-level
    placement-algorithm verification (already covered by A5 / A6 /
    A7), NOT A10-layout. A10-layout requires real-world physical
    placement vs the algorithm's prediction.
  - (b) Defer A10-layout silently and retroactively redefine A10 as
    geometry-only — rejected; the strategic report scorecard would
    flag this as goal-post-moving.
  - (c) Mark A10-layout DEFERRED-with-classification (v0.9 chosen)
    — accepted; preserves the original v0.1 north star without
    inventing fake substitutability.
- Why chosen — honest classification > silent goal-shifting; future
  v0.10+ A10b in-situ work continues to carry both A10a (geometry)
  and A10-layout responsibility.
- Consequences:
  - The strategic-position scorecard (§3 row A10) henceforth reads
    "A10a PASS (substitute), A10b FAIL (deferred), A10-layout
    DEFERRED (classified non-substitutable)".
  - v0.9 release notes explicitly carry the three-way decomposition.
  - Reverse-trigger: if a public dataset eventually ships VBAP layout
    GT (unlikely but possible), ADR 0017 is reverted and an A10a-
    layout substitute is added in v0.10+.
- References (v0.8 strategic-position report; ADR 0016; v0 design
  Q9 / A10).
