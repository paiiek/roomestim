## Findings confirmed at HEAD (read, not re-derived)

- Predictor: `roomestim/reconstruct/predictor.py:461` builds `base_walls = [s for s in room.surfaces if s.kind == "wall"]`, passes to `_objects_to_wall_alpha_overrides` which indexes `walls[wall_idx]` (lines 295–301). **Walls-only frame.**
- Viewer: `roomestim_web/viewer.py:250` does `room.surfaces[obj.wall_index]`. **Full-surfaces frame.** ← the bug.
- Adapter surface order: `[floor, *ceilings, *walls]` (`roomestim/adapters/roomplan.py:294`). So `wall_index=0` → wall0 in predictor, floor in viewer.
- Schema: `proto/room_schema.v0_2.draft.json:92` defines `wall_index` as bare `integer ≥ 0` / null — NO frame documented.
- Mesh: `roomestim/adapters/mesh.py:155` emits `"0.1-draft"`; RoomPlan (`roomplan.py:308`) and `RoomModel` default emit `"0.2-draft"`.
- **Mesh output is NOT jsonschema-validated anywhere** — `mesh.py` has no jsonschema import; the 0.1 schema (`room_schema.draft.json`) is referenced only by `export/room_yaml.py` and `io/room_yaml_reader.py`, not by the adapter. So emitting `"0.2-draft"` from mesh.py is a pure label change with no validation consequence.
- Only ONE test asserts mesh = 0.1-draft: `tests/test_adapter_mesh.py:40`. The 0.1-draft backward-parse tests (`test_objects.py`, `test_schema_backward_parse.py`, `test_schema_stage2_validates.py`, `test_export_*`) all exercise the **reader/exporter**, not the mesh adapter — unaffected.

### T4 — Regression test for ① (THE load-bearing proof)
A door/window at a **nonzero** `wall_index` makes floor-vs-wall divergence observable. Add one shared assertion across both lanes:

- **Predictor side — new `tests/test_wall_index_frame.py`:** build a shoebox via `RoomPlanAdapter().parse(lab_room.json, octave_band=True)` (or `evolve_room_add_object`) with a door at `wall_index=2`. Assert `_objects_to_wall_alpha_overrides(objects, base_walls)` resolves to `walls[2]` (the same wall the viewer must pick), and that the override applies to a `kind=="wall"` surface, never floor/ceiling.
- **Viewer side — new `tests/web/test_wall_index_viewer.py`:** same room + door at `wall_index=2`; call `_wall_attached_traces`; assert the produced quad lies on `walls[2].polygon` (e.g. its `base` point matches `walls[2].polygon[0]` projected), i.e. resolves to the identical wall surface the predictor uses. Add a guard asserting the wall it picks is `kind=="wall"` (would FAIL on the old `room.surfaces[2]` = a wall-or-ceiling depending on count, and FAIL on `wall_index=0` → floor).
- **Shared invariant:** both tests assert `room.surfaces.index(picked) == ([s for s in room.surfaces if s.kind=="wall"]).index something` consistent — i.e. predictor-picked wall IS viewer-picked wall.

**Acceptance:** both new tests FAIL on HEAD viewer code (pre-fix) and PASS post-T1. They are the regression lock.

## ③ / ④ Scope Recommendation

### ③ `CaptureAdapter` Protocol repair — **RECOMMEND: IN (low risk, high value).**
- `base.py:25` Protocol omits `octave_band`; all 3 adapters (`mesh:47`, `roomplan:194`, `polycam:31`) already accept it. CLI works around it with `getattr(adapter, "parse")` at `cli.py:347/460` purely to dodge the Protocol mismatch.
- **Fix:** add `octave_band: bool = False` to the Protocol `parse` signature (`base.py:25`), then change `cli.py:347` and `cli.py:460` from `parse = getattr(adapter, "parse"); room = parse(...)` to direct `room = adapter.parse(args.input, scale_anchor=None, octave_band=octave_band)`.
- **Risk:** LOW. Restores static typing on the hottest call path; no runtime behavior change (the methods already have the param). Grep found NO `# type: ignore` currently in cli.py — the `getattr` was the suppression mechanism; removing it strengthens mypy coverage. Verify mypy strict still 0 after. ~3 line edits across 2 files. Folds cleanly into "adapter-consistency" theme of this cycle.

### ④ Hardcoded engine-schema absolute path — **RECOMMEND: OUT (defer to v0.20, open OQ).**
- `layout_yaml.py:57–59` `_DEFAULT_ENGINE_SCHEMA_PATH` falls back to an absolute `/home/seung/...` path when neither `SPATIAL_ENGINE_REPO_DIR` nor `--validate-engine` is set.
- **Blast radius is REAL:** `tests/test_export_layout_yaml.py:29–42` and `tests/test_engine_toggle.py` (160–288) actively depend on this fallback (they replicate `_DEFAULT_ENGINE_SCHEMA_PATH` and exercise ENV/CLI precedence per D42/ADR 0033). Replacing the fallback with a hard error changes documented runtime behavior (cli.py:141/260 help text promises "the hardcoded default when omitted") and would require coordinated test + help-text + ADR 0033 amendments.
- **Why defer:** it is NOT a correctness bug — it is a portability/UX wart, orthogonal to ①/②/③. Bundling it inflates this tight cycle and entangles a behavior change with a pure-fix release. Open **OQ-42** ("replace hardcoded engine-schema fallback with explicit error or vendored default; touches ADR 0033 + cli help + 2 test files"), cadence v0.20.

**Net committed scope: ① + ② + ③.** ④ deferred via OQ.

## Version Bump — **RECOMMEND MINOR (v0.18.5 → v0.19.0).**
- **Not PATCH:** ① is a user-visible **behavior correction** in viewer geometry (wall-attached door/window quads previously rendered against the wrong surface for any nonzero `wall_index`). Under SemVer, a bug fix is PATCH-eligible, BUT ② changes the `schema_version` string emitted by `MeshAdapter` from `0.1-draft` to `0.2-draft` — a change in **observable output of a public adapter**. Consumers keying on that string see different output. That is the SemVer line for MINOR (additive/behavioral output change, backward-compatible since 0.2-draft is a superset and the reader already accepts both).
- **Not MAJOR:** no breaking API removal; 0.2-draft reader accepts old 0.1-draft input (backward parse intact).
- **Anti-inflation note:** this is a genuine correctness + output-contract change backed by code edits and a regression test — not a doc-only churn bump like the recent v0.18.1–v0.18.5 PATCH line. The MINOR is justified by the mesh schema_version output change specifically; if ② were dropped, this would correctly be a PATCH. If you prefer to keep ② out and ship ① + ③ only, downgrade to **v0.18.6 PATCH**.
