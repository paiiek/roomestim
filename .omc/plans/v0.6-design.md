## §0a Locked-scope precedent (mirrors v0.5-design.md §0a style)

The TASLP-MISC headline was selected by the v0.6 orchestrator brief and validated
by the planner against three constraints:

1. **It directly tests v0.4 Finding 4 hypothesis** ("bare-walls model under-counts
   absorption budget — soft furnishings/occupants"). v0.5.0 reserved `MISC_SOFT`
   as a schema slot (ADR 0011); v0.5.1 retroactively justified the slot by citing
   TASLP §II-C explicit furniture counts (D16 §"MISC_SOFT retroactive
   justification"). v0.6 wires the slot to the adapter so the hypothesis becomes
   empirically testable on the next gated E2E run.
2. **It is structurally additive**. No existing default-lane test imports the
   adapter. The change is a new helper plus one additional `Surface` per ACE
   room synthesised by `_build_room_model(...)` (now indirectly via
   `_room_case_for(...)` / `load_room(...)`). Default-lane byte-equality (84
   tests) is preserved by construction.
3. **It uses canonical TASLP §II-C furniture counts** (durable in
   `project_taslp_2016_content.md`). Furniture counts are factual data (not
   copyrightable); per-piece α values are cited from textbooks (fair-use), not
   transcribed wholesale.

**Locked decision summary** (full rationale per locked-decision in §3 / §4):

| Lock | Choice | Rationale |
| --- | --- | --- |
| Headline | TASLP-derived per-room MISC_SOFT surface budget | Direct test of v0.4 F4 hypothesis; uses canonical TASLP §II-C counts |
| Source for per-piece α | **Vorländer 2020 *Auralization* §11 / Appx A** primary; Beranek 2004 *Concert Halls and Opera Houses* Ch.3 cross-checked for lecture-seat values | Vorländer is roomestim's existing citation anchor (D7, D12, M1); Beranek is the auditorium-standard for theatre seating |
| Building_Lobby | **EXCLUDED** from MISC_SOFT budget | TASLP §II-C: irregular shape + coupled spaces; shoebox model violation already documented in ADR 0012 — adding furniture absorption to a non-shoebox room compounds modelling error |

**Why TASLP-MISC and only TASLP-MISC for the headline**: it is the smallest
self-contained additive change that empirically tests an open hypothesis (v0.4
F4) using a canonical source (TASLP §II-C). All other candidates either need
non-canonical evidence (lab visit / author email) or are explicitly deferred by
existing decisions (D8, D11, D14, ADR 0008, ADR 0009).

room with `MaterialLabel.MISC_SOFT`. It does NOT touch existing wall/ceiling
material strings (which remain INDETERMINATE per ADR 0012). The adapter is now
emitting two kinds of best-guess: (a) walls/ceiling strings that pre-date
TASLP review, and (b) MISC_SOFT surface area derived from canonical counts ×
textbook per-piece α. Category (b) is *more* canonical than (a), not less.

```python
def _furniture_to_misc_soft_area(furniture: dict[str, int]) -> float:
    """Return synthetic MISC_SOFT surface area (m²) for a furniture inventory.

    The adapter does NOT model individual chairs as separate Surface objects.
    Instead it emits one synthetic MISC_SOFT surface whose AREA is chosen so
    that the Sabine integrand `area * MaterialAbsorption[MISC_SOFT]` equals
    the sum of per-piece equivalent absorption (Σ count_i * A_i).

    Formula (preserves Sabine integrand):
        area_misc_soft = Σ_pieces count_i * A_500_i / a_misc_soft_500

    With `MaterialAbsorption[MISC_SOFT] == 0.40` (v0.5.0 row), this scales the
    per-piece sum to a notional surface area at the 500 Hz reference band.
    Per-band Sabine and Eyring then operate per the MISC_SOFT band tuple
    (which itself is representative-not-verbatim; see model.py).

    Parameters
    ----------
    furniture:
        Mapping `piece_name -> count`. Piece names must be keys of
        `_PIECE_EQUIVALENT_ABSORPTION_500HZ_M2`.

    Returns
    -------
    float
        Synthetic MISC_SOFT surface area in m² (≥ 0).

    Raises
    ------
    KeyError
        If any piece in `furniture` is not in
        `_PIECE_EQUIVALENT_ABSORPTION_500HZ_M2`.
    """
    from roomestim.model import MaterialAbsorption, MaterialLabel

    a_misc_soft = MaterialAbsorption[MaterialLabel.MISC_SOFT]
    # Defensive — matches the v0.5.0 enum row; should never be 0
    if a_misc_soft <= 0.0:
        raise ValueError("MaterialAbsorption[MISC_SOFT] must be > 0")
    total_eq_abs = 0.0
    for piece, count in furniture.items():
        a_i = _PIECE_EQUIVALENT_ABSORPTION_500HZ_M2[piece]
        total_eq_abs += count * a_i
    return total_eq_abs / a_misc_soft
```

**Acceptance criteria for §2.2**:

- `KeyError` raised for unknown piece string (test in §2.5).
- For Office_1 (`{"office_chair": 4}`):

```python
def _misc_soft_surface_from_furniture(
    room_id: str,
    L: float,
    W: float,
) -> Surface | None:
    """Build a synthetic MISC_SOFT Surface for `room_id`, or None if absent.

    The synthetic surface has a square polygon of side √area positioned in the
    floor plane (y=0) at an arbitrary corner offset; the polygon coordinates
    are NOT geometrically meaningful — only the polygon area is consumed by
    `_surface_areas_by_material(...)` for Sabine / Eyring.

    Returns None when `room_id` is not in `_FURNITURE_BY_ROOM` (e.g.
    Building_Lobby per §3 OQ-9). The caller (`_build_room_model`) skips
    appending a None.

    The polygon is sized so that the Newell-area formula in
    `_surface_areas_by_material` returns the requested area. To stay inside the
    floor footprint and avoid intersecting other surfaces (visually; physics is
    unaffected), the polygon is clipped to fit inside `L × W` if the requested
    side > min(L, W). When clipped, the surface is laid out as a thin strip
    along the longer floor edge so the integrand is preserved.
    """
    from roomestim.model import MaterialAbsorption, MaterialLabel
    import math

    if room_id not in _FURNITURE_BY_ROOM:
        return None
    furniture = _FURNITURE_BY_ROOM[room_id]
    area = _furniture_to_misc_soft_area(furniture)
    if area <= 0.0:
        return None

    # Decide polygon shape: square of side √area, clipped to floor footprint.
    side = math.sqrt(area)
    if side <= min(L, W):
        # Square inside floor, near origin corner
        x0, x1 = 0.0, side
        z0, z1 = 0.0, side
    else:
        # Strip along longer edge
        long_side = max(L, W)
        short_side = min(L, W)
        strip_long = min(area / short_side, long_side)
        if L >= W:
            x0, x1 = 0.0, strip_long
            z0, z1 = 0.0, area / strip_long
        else:
            x0, x1 = 0.0, area / strip_long
            z0, z1 = 0.0, strip_long

    band_tuple = MaterialAbsorptionBands[MaterialLabel.MISC_SOFT]
    return Surface(
        kind="floor",  # adapter convention — surface lies in floor plane
        polygon=[
            Point3(x0, 0.0, z0),
            Point3(x1, 0.0, z0),
            Point3(x1, 0.0, z1),
            Point3(x0, 0.0, z1),
        ],
        material=MaterialLabel.MISC_SOFT,
        absorption_500hz=MaterialAbsorption[MaterialLabel.MISC_SOFT],
        absorption_bands=band_tuple,
    )
```

**Acceptance criteria for §2.3**:

```python
def test_misc_soft_surface_office1_area_preserves_integrand():
    """Office_1 MISC_SOFT surface polygon area equals helper result within 1e-9."""
    geom = ACE_ROOM_GEOMETRY["Office_1"]
    L, W = float(geom["L"]), float(geom["W"])
    surf = _misc_soft_surface_from_furniture("Office_1", L, W)
    assert surf is not None
    assert surf.material == MaterialLabel.MISC_SOFT
    expected = _furniture_to_misc_soft_area(_FURNITURE_BY_ROOM["Office_1"])

    # Newell area on the surf.polygon (mirrors _surface_areas_by_material)
    pts = surf.polygon
    n = len(pts)
    nx = ny = nz = 0.0
    for i in range(n):
        j = (i + 1) % n
        nx += (pts[i].y - pts[j].y) * (pts[i].z + pts[j].z)
        ny += (pts[i].z - pts[j].z) * (pts[i].x + pts[j].x)
        nz += (pts[i].x - pts[j].x) * (pts[i].y + pts[j].y)
    actual = 0.5 * math.sqrt(nx * nx + ny * ny + nz * nz)
    assert actual == pytest.approx(expected, abs=1e-9)
```

## §3 Open questions log mirror (OQ-6..OQ-10)

These five OQs are appended to `.omc/plans/open-questions.md` per planner
protocol.

- **OQ-6** Per-piece α table source — Vorländer 2020 *Auralization* §11 / Appx A
  vs Beranek 2004 *Concert Halls and Opera Houses* Ch.3 Table 3.1.

  Meeting_1 / Meeting_2). *Why it matters*: meeting-room chairs are typically
  closer to office chairs (movable, padded back) than to fixed theatre
  seating, but textbook tables vary. *Locked default*: Meeting_1 / Meeting_2
  → `"office_chair"` (per §0a Lock table). Reverse if the executor finds a
  Vorländer §11 "meeting chair" row that justifies a separate piece type.

- **OQ-8** Plumbing — direct in `_room_case_for` factory (no separate helper)
  vs separate helper. *Why it matters*: single-call-site simplicity vs
  unit-testability. *Locked default*: separate helper
  (`_misc_soft_surface_from_furniture`). Reverse if the helper has only one
  call site by ship date and adds no test surface (it does — see §2.5; the
  default holds).

- **OQ-9** Building_Lobby treatment in v0.6 — (a) exclude (cleanest), (b)
  per-area MISC_SOFT density, or (c) defer to separate ADR. *Why it matters*:
  Building_Lobby is a coupled-space non-shoebox per TASLP §II-C / ADR 0012;
  adding a MISC_SOFT budget compounds modelling error.
  *Locked default*: **(a) EXCLUDE**. v0.6 ships with `Building_Lobby` absent
  from `_FURNITURE_BY_ROOM`; the surface synthesis helper returns `None`.
  ADR 0013 names this as the v0.6 default; a future ADR (v0.7+) re-evaluates
  Building_Lobby coupled-space modelling on its own terms (geometry kind
  annotation, multi-room volume, etc.). Reverse only if a coupled-space ADR
  ships first.

- **OQ-10** New `MaterialLabel.FLOOR_HARD` enum entry for hard-floored rooms
  with unspecified subtype (Lecture_1 / Lecture_2 / Building_Lobby).
  *Why it matters*: TASLP §II-C says "hard-floored" but doesn't specify wood
  / concrete / tile / linoleum; current `wood_floor` is a HARD-FLOOR-COMPATIBLE
  best-guess (D16). Adding `FLOOR_HARD` would make the indeterminacy explicit
  in the schema.
  - **Pro**: makes the indeterminacy schema-visible; prevents future readers
    assuming `wood_floor` is verified.
  - **Con**: schema-impacting (Stage-1 absorbs gracefully but Stage-2 flip
    would carry it permanently); requires a new `MaterialAbsorption` row +
    `MaterialAbsorptionBands` row that are themselves representative-not-
    verbatim (we'd be adding *more* indeterminacy to the table without a
    measurement); the honesty-caveat block in `ace_challenge.py` already
    documents the `wood_floor` HARD-FLOOR-COMPATIBLE state per D16.
  *Locked default*: **NO new enum entry in v0.6**. The existing v0.5.1
  honesty caveat is sufficient. Reverse if (a) hard-floor subtype is later
  confirmed for any room (lab visit / author email) AND (b) the confirmed
  subtype is not already in the enum.
