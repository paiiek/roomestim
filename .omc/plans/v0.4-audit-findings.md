## Finding 1 — Local corpus has no machine-readable room metadata

The local distribution at `/tmp/ace_corpus/` contains:
- `ACE_Corpus_instructions_v01.pdf` — high-level dataset use instructions; no per-room
  geometry / material breakdown.
- `software/Software/private/getACECorpusData.m` — MATLAB private helper. Grep for
  `corpusData.ROOMS` returns the codename → friendly-name table (transcribed verbatim
  below; do not paraphrase).
- The per-room L×W×H + material breakdown that `ACE_ROOM_GEOMETRY` encodes is
  **NOT** present in any local file. The canonical published source is
  Eaton 2016 TASLP Table I.

**Codename → friendly-name mapping** (transcribed verbatim from
`software/Software/private/getACECorpusData.m` `corpusData.ROOMS`, lines 272–280):

| Codename | Friendly name |
| --- | --- |
| `803` | Office 2 |
| `503` | Meeting Room 1 |
| `611` | Meeting Room 2 |
| `508` | Lecture Room 1 |
| `403a` | Lecture Room 2 |
| `502` | Office 1 |
| `EE_Lobby` | Building Lobby |

The `.m` source comment notes "502 and EE_Lobby included for completeness, although
they never actually get used [in the EVAL decode path]"; this note is about the EVAL
algorithm's room set, not the corpus measurement set — all 7 rooms are present in
the per-room T60 CSVs the v0.3.0 ACE adapter consumes.

**Status**: byte-cross-check of `ACE_ROOM_GEOMETRY` against Eaton 2016 Table I is
**DEFERRED**. The current values were transcribed from corpus README documentation
during v0.3 (see existing honesty caveat in `roomestim/adapters/ace_challenge.py`
module docstring). They have NOT been verified against the canonical
table. Re-open this finding once the paper is available.

## Finding 2 — Eyring shipped; not the dominant error source

The `eyring_rt60` and `eyring_rt60_per_band` predictors are now available alongside
Sabine. The gated E2E test reports both per room and per band, with the
`eyring ≤ sabine + 1e-9` invariant enforced at runtime (Vorländer 2020 §4.2).

  the ranking of "which rooms are hardest"; the dominant error source is elsewhere.

## Finding 3 — Lecture_2 ceiling-material hypothesis (UNCONFIRMED)

The Lecture_2 room currently maps `"ceiling": "ceiling_acoustic_tile"` in
`ACE_ROOM_GEOMETRY`. With α₅₀₀ = 0.55, this gives a high ceiling absorption budget,
which **under-predicts** RT60 (Sabine 500Hz ≈ 0.673 s vs measured 1.343 s; Eyring
0.593 s — Eyring widens the gap because Eyring is strictly ≤ Sabine).

**Hypothesis**: the actual ceiling is painted plaster / drywall
(`ceiling_drywall`, α₅₀₀ = 0.10). With this swap, total absorption drops by
≈(0.55 − 0.10) × ceiling_area, and the Sabine RT60 rises into the measured range.

**Status**: HYPOTHESIS ONLY. Cannot be confirmed without Eaton 2016 Table I (Finding 1).
DEFERRED to v0.5 ADR.

## Finding 4 — Bare-walls model under-counts absorption budget

Building_Lobby (+1.425 s err Sabine / +1.369 s Eyring), Lecture_1 (+1.201 / +1.131),
Office_1 (+0.486 / +0.438), and Office_2 (+0.435 / +0.385) all over-predict at 500 Hz.
The error sign is consistent with the bare-walls 6-surface model missing the absorption
contribution from soft furnishings (chairs, desks, fabric panels, books, projection
screens, curtains) and any occupants present during the corpus measurement. Sabine
attributes 100% of absorption to the 6 wall/floor/ceiling surfaces; the actual
absorption budget is larger.

**Two follow-ups, BOTH DEFERRED to v0.5 ADR**:
- (a) Revise `MaterialAbsorptionBands` for `wall_painted`, `wood_floor`,
  `ceiling_drywall` — only justified if Finding 3's hypothesis is rejected, otherwise
  Finding 3 alone explains the magnitude on at least Lecture_2.
- (b) Extend `MaterialLabel` enum with `MISC_SOFT` (or similar) to let adapters
  account for furnishings as a separate budget line. This is a schema-impacting change
  and needs its own ADR + Stage-2 schema discussion.

## Status update 2026-05-07 (post-v0.5.0; framing correction at v0.5.1)

The Eaton 2016 TASLP final paper (DOI 10.1109/TASLP.2016.2577502) was obtained
via SNU IEEE Xplore on 2026-05-07. The earlier framing in this file ("the
canonical published source is Eaton 2016 TASLP Table I" for the per-room
material breakdown) is **incorrect**. Reviewed cover-to-cover, the TASLP final:

- **Table I (p.1683)** lists per-room L, W, H, Volume, mean FB T60, mic-source
  distances, and DRR. Byte-identical to arXiv:1606.03365 Table 1 (used as the
  source of truth at v0.5.0; ADR 0010 stands). **No material columns.**
- **§II-C "Rooms" (p.1683)** gives a one-sentence prose description per room
  with floor type ("carpeted" / "hard-floored") + furniture counts. **No walls,
  no ceiling.** The full per-surface material assignment table that v0.4
  audit framed as "the canonical source" does **not** exist in the published
  paper.

### What this changes

- **Finding 1 — materials half**: status changes from `DEFERRED (TASLP-blocked)`
  to `INDETERMINATE (canonical source does not contain it)`. No future research
  route should propose "wait for TASLP" for walls / ceiling material data.
- **Finding 1 — floors half**: 4/7 rooms BYTE-CONFIRMED at TASLP §II-C
  ("carpeted" → roomestim `floor: carpet`): Office_1, Office_2, Meeting_1,
  Meeting_2. 3/7 rooms HARD-FLOOR-COMPATIBLE but unspecified subtype:
  Lecture_1, Lecture_2, Building_Lobby ("hard-floored" — wood / concrete / tile
  not distinguished by the paper).
- **Building_Lobby**: TASLP §II-C says "large irregular-shaped hard-floored
  room with coupled spaces including a café, stairwell and staircase.
  Measurements in Table I correspond to the corner area where the recordings
  were made whereas the total volume of the lobby is many times larger." This
  is a structural caveat: `ACE_ROOM_GEOMETRY["Building_Lobby"]` shoebox of
  72.9 m³ describes the recording corner, not the room. Sabine RT60 prediction
  on a non-shoebox coupled space is not expected to match measurement; the
  v0.4 perf doc's +1.425 s err Sabine on Building_Lobby is consistent with a
  modelling-assumption violation, not (only) a coefficient/material gap.
- **Finding 3 (Lecture_2 ceiling)**: status stays `HYPOTHESIS` but the upstream
  evidence path is now closed-with-no-result. Promoting to DONE or REJECTED
  requires a different source (lab visit photos, author email, on-site
  inspection); the canonical paper is exhausted.
- **Finding 4 (bare-walls under-counts absorption)**: TASLP §II-C explicit
  furniture counts (Office_2 "6 chairs + bookcase"; Lecture_2 "~100 chairs +
  ~35 tables"; etc.) are now the direct empirical motivation for the v0.5
  `MISC_SOFT` enum slot (F4b shipped). Per-room MISC_SOFT surface area can
  be derived from these counts via Beranek 2004 / Vorländer 2020 per-piece
  equivalent absorption — out-of-scope for v0.5.x but available for v0.6+.

### What stays unchanged

- v0.5.0 dimensional patch (Office_2 W 3.50 → 3.22, H 3.00 → 2.94) and the
  ADR 0010 "verified vs arXiv:1606.03365 Table 1" framing are **vindicated by
  the TASLP final** (TASLP Table I = arXiv 1606.03365 Table 1 byte-identical).
  No code change needed.
- v0.5.0 `MISC_SOFT` enum (ADR 0011) is retroactively strengthened by TASLP
  furniture counts, not weakened.
- Wall and ceiling material strings in `ACE_ROOM_GEOMETRY` remain best-guess;
  honesty caveat in `roomestim/adapters/ace_challenge.py` is updated at v0.5.1
  to reflect "TASLP final reviewed; materials not in paper" so future readers
  do not repeat the v0.4 framing.
