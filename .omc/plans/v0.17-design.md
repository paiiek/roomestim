## 3. 구현 Phase (high-level)

### Phase 1 — Schema bump
- `roomestim/proto/room_schema.draft.json` 업데이트 — `objects` 배열 추가
- `Object` dataclass NEW: `kind: Literal["column", "door", "window"]`, `polygon_2d: list[Point2]`, `height_m: float`, `material: MaterialLabel`, `absorption_*`
- `roomestim/model.py` `RoomModel` 확장: `objects: list[Object] = field(default_factory=list)`
- Backward parse: 구 schema `0.1-draft` parsing은 `objects=[]` 자동

### Phase 2 — Adapter object 인식
- RoomPlanAdapter: RoomPlan JSON `Object` 카테고리 매핑 (column/door/window)
- MeshAdapter: 사용자 수동 마크업 (이번 release에서는 placeholder)
- ACEChallenge: objects=[]

### Phase 3 — Web UI 객체 추가
- "Add Column" 모드: 2D 블루프린트(v0.16)에서 클릭 → column 추가
- 마찬가지로 "Add Door" / "Add Window"
- evolve_room patch — `evolve_room(room, *, objects=...)`

### Phase 4 — Acoustic 통합
- `predictor.py` extension: ISM α 계산 시 column 표면 추가
- Door/Window는 별도 α (door=`wall_painted` 비슷, window=`glass`)
- ADR 0009 invariant 회귀 lock

### Phase 5 — USDZ + gLTF export
- `roomestim/export/usd.py` NEW — `pyusd` 의존, RoomModel → UsdGeom.Mesh + 재질 sub-prim
- `roomestim/export/gltf.py` NEW — `trimesh` 활용 (이미 dep)
- CLI: `roomestim export --format usdz | gltf | yaml`

### Phase 6 — 테스트 + 문서
- `tests/test_objects.py` NEW
- `tests/test_usdz_export.py`, `tests/test_gltf_export.py` NEW
- `tests/web/test_object_add_ui.py` NEW
- ADR 0032/0033 작성
- README §객체 추가 / §USDZ export 절 NEW
