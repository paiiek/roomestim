## 1. 목표 — Fix 7b closure (CLI el-bound enforcement) — 신규 ADR 없음

v0.18.0 이 명시 deferral 한 단일 known-gap 을 닫는다: `nudge_speaker` 의
spherical 분기가 산출한 `el2 = degrees(el_rad) + del_deg` 가 [-90, 90] 밖이면
**거부**한다 (`ValueError`). 이 변경은 core-only (`roomestim/edit.py` +
`roomestim/cli.py` 1-line help note). ADR 0036 §B/§C 본문 byte-equal (nudge
정책의 *보강*이지 정책 변경 아님 — finite 검사가 막던 NaN/inf 에 더해
non-physical el 도 막는 것). web lane byte-equal.

### 1.1 Root cause (검증됨)

`roomestim/edit.py::nudge_speaker` 의 spherical 경로 (line 434–442):

```python
az_rad, el_rad, dist = cartesian_to_pipeline(p.x, p.y, p.z)
az2 = math.degrees(az_rad) + daz_deg
el2 = math.degrees(el_rad) + del_deg           # ← el2 가 [-90, 90] 밖일 수 있음
d2 = dist + ddist_m
if d2 <= 0.0:
    raise ValueError(f"nudge_speaker: resulting dist {d2} must be > 0")
nx, ny, nz = yaml_speaker_to_cartesian(az2, el2, d2)   # ← el2>90 → cos(el2)<0
new_pos = Point3(x=nx, y=ny, z=nz)
```

`yaml_speaker_to_cartesian(az2, el2, d2)` (`roomestim/coords.py:56-71`) 는
`el ∈ (90, 270)` 에서 `cos(el) < 0` → x/z hemisphere 가 뒤집히고 y 는 climb
후 descend 하는 **비물리 좌표** 를 산출한다 (모두 finite 이므로
`evolve_placement` 의 `assert_finite` 가 통과시킴 — line 358–365). 즉
"el=80 인 스피커를 +20° nudge" → el2=100 → 사용자 의도는 "더 위로"인데 결과
좌표는 거울 반사된 위치. web 은 `gr.Number(minimum=-90, maximum=90)` 로
입력 단계에서 차단되나 (`roomestim_web/speaker_nudge.py`), CLI (`roomestim
edit --del-deg`) 는 임의 float 허용 → 무방비.

## 2. 결정 (신규 D 1건 [D53]; 신규 ADR 없음; ADR 0036 + ADR 0030 §Status-update append)

### 2.1 핵심 행위 결정 — clamp vs reject (D53)

**선택: REJECT (`ValueError`).** spherical 분기에서 `el2 ∉ [-90, 90]` 이면
`yaml_speaker_to_cartesian` 호출 **전에** `ValueError` 를 raise 한다.

**근거 (reject 채택):**
1. **기존 error 계약과의 일관성 (강한 precedent).** 동일 함수 `nudge_speaker`
   가 이미 두 조건을 `ValueError` 로 거부한다: (i) frame mixing (line 426–429),
   (ii) `d2 <= 0` 비물리 거리 (line 439–440). el ∉ [-90, 90] 는 d2 <= 0 와
   **동급의 비물리 입력** — 같은 frame(spherical)에서 같은 종류의 "결과가
   물리적으로 무의미" 게이트이므로 같은 mechanism(ValueError) 이 일관적.
2. **호출자 체인이 이미 reject 를 처리한다.** CLI `main()` 은
   `except (ValueError, OSError, RuntimeError, IndexError)` 로 잡아
   `error: <msg>` → stderr + `return 1` (cli.py:559–561) — 추가 wiring 불필요.
   web `_on_nudge_speaker` 는 이미 `nudge_speaker` 를 try/except ValueError 로
   감싸 빨간 status 표시 (v0.18 design `:657-659`) — 변경 불필요. clamp 였다면
   양쪽 호출자가 "값이 조용히 바뀌었음" 을 사용자에게 알릴 별도 경로가 필요.
3. **사용자 의도 보존.** "el=80 스피커를 +20° nudge" 요청은 *개념적 오류*
   (반구 경계 초과)이지 truncate 할 값이 아니다. clamp(→90)는 의도를 silently
   왜곡하며, 사용자는 "왜 +20 했는데 +10만 갔지?" 로 혼란. reject 는 명시적
   에러 메시지(`resulting elevation {el2}° outside [-90, 90]`)로 즉시 피드백.
4. **non-idempotency 회피.** clamp 면 같은 `--del-deg 20` 을 반복 적용 시 90°에
   saturate — 멱등성 깨짐 + diff 가 "변경 없음" 으로 나오는 surprising UX.
   reject 면 매 호출이 명확히 성공 또는 실패.
5. **web 제약과의 정합성.** web `gr.Number(minimum=-90, maximum=90)` 은
   *입력 제약* (사용자가 애초에 91 을 입력 못함)이지 *값 변형* 이 아니다.
   "UI 가 막고 core 가 강제한다" 는 reject 와 정합 — clamp 는 "UI 는 막고 core 는
   조용히 고친다" 로 두 레이어의 의미가 어긋남.

**거부된 대안 (Rejected alternatives):**
- **(A) Clamp el to [-90, 90].** nudge-UX friendliness (경계 넘겨도 "그냥 끝까지
  올림") 가 장점이나 §2.1 의 1·3·4·5 근거로 거부. 특히 기존 `dist <= 0` 가
  clamp(→ε) 가 아니라 reject 인 것과 비대칭이 됨 — 같은 함수 내 두 비물리
  게이트가 다른 정책을 쓰는 것은 audit-trail 일관성 저하.
- **(B) Silently accept (현행 v0.18.0).** Fix 7b 가 닫으려는 바로 그 버그 —
  거부.
- **(C) Warn-and-clamp (logging.warning + clamp).** core 라이브러리가 stdout/
  stderr 로 warning 을 흘리는 것은 CLI diff 출력을 오염시키고 (cli.py 가
  `sys.stdout.writelines(diff)` 직후 print), 라이브러리 호출자에게 부작용.
  reject 가 부작용 0 + 명시적.

**Reverse-criterion (D53):** 사용자가 "el 경계 넘어가도 nudge 끝까지 적용(clamp)"
을 ≥ 2회 요청 → `nudge_speaker(..., clamp_el: bool = False)` opt-in clamp 파라미터
추가 후 ADR 0036 §Status-update. default 는 reject 유지 (안전 default).

### 2.2 Cartesian 분기는 unguarded 로 유지 (명시 결정)

Cartesian 분기 (line 432–433: `new_pos = Point3(x=p.x+dx, y=p.y+dy, z=p.z+dz)`)
는 **el 을 직접 산출하지 않는다** — 임의의 finite (x, y, z) 는 항상 유효한 좌표
이며 그에 대응하는 el = `atan2(y, sqrt(x²+z²))` 는 정의상 [-90, 90] 안에 든다
(`cartesian_to_pipeline` 의 atan2 결과는 [-π/2, π/2]). 즉 Cartesian nudge 는
"비물리 el" 을 만들 수 없다 — 결과 위치가 함의하는 el 은 언제나 물리적. 따라서
**Cartesian 분기는 가드하지 않는다** (추가 검사 = dead guard). 이 결정을 D53
본문 + ADR 0036 §Status-update-v0.18.1 에 명시하여 "왜 한쪽만 가드하나" 를
audit-trail 에 남긴다.

(주: Cartesian Δ 로 스피커를 origin 정확히로 옮기면 dist=0 → 후속 spherical
nudge 시 el = atan2(0, 0) = 0 (Python `math.atan2(0,0)=0.0`, finite) 로 안전.
별도 가드 불필요 — `cartesian_to_pipeline` 은 dist=0 에서도 finite 반환.)

### 2.3 가드 위치 — spherical 분기, el2 계산 직후

가드는 `nudge_speaker` 의 spherical `else` 분기 안, `el2 = ...` 다음 + `d2 <= 0`
게이트와 함께 둔다 (둘 다 spherical 비물리 게이트). 함수 docstring 의 `Raises`
절 `ValueError` 항목에 "or when the resulting elevation is outside [-90, 90]
degrees (non-physical)" 추가. `evolve_placement` 의 finite 검사는 **변경 없음**
(NaN/inf 차단 책임 유지; el-bound 는 nudge_speaker 의 spherical-frame 책임 —
관심사 분리).

## 3. 구현 (Phase 1 → Phase 4 순차)

### Phase 1 — `roomestim/edit.py::nudge_speaker` el-bound guard (D53)

- spherical `else` 분기 (현 line 434–442) 수정:
  ```python
  else:
      az_rad, el_rad, dist = cartesian_to_pipeline(p.x, p.y, p.z)
      az2 = math.degrees(az_rad) + daz_deg
      el2 = math.degrees(el_rad) + del_deg
      # v0.18.1 (D53 / Fix 7b closure): reject non-physical elevation. el ∉
      # [-90, 90] would flip the x/z hemisphere via cos(el)<0 in
      # yaml_speaker_to_cartesian — finite but mirror-reflected (a silent
      # hemisphere flip). Mirrors the existing dist<=0 reject (same frame,
      # same class of non-physical input). Cartesian branch needs no guard:
      # any finite (x,y,z) implies a physical el ∈ [-90,90] via atan2.
      if not (-90.0 <= el2 <= 90.0):
          raise ValueError(
              f"nudge_speaker: resulting elevation {el2}° outside [-90, 90] "
              f"(non-physical); reduce del_deg or use a Cartesian Δ"
          )
      d2 = dist + ddist_m
      if d2 <= 0.0:
          raise ValueError(f"nudge_speaker: resulting dist {d2} must be > 0")
      nx, ny, nz = yaml_speaker_to_cartesian(az2, el2, d2)
      new_pos = Point3(x=nx, y=ny, z=nz)
  ```
  - 경계 포함 정책: `[-90, 90]` **inclusive** (el=±90 는 물리적 천정/바닥 —
    `yaml_speaker_to_cartesian(az, 90, d)` = (0, d, 0) finite + 물리적). 따라서
    `not (-90.0 <= el2 <= 90.0)` (el2=90.0 통과, el2=90.0000001 거부).
  - 부동소수 경계: el_rad 가 atan2 산출이므로 정확히 90.0 인 입력 (el=0 + del_deg
    =90) 은 `90.0 <= 90.0` True → 통과. v0.18 design `:831` 의 el=89 boundary
    케이스 (이미 test 존재 — `test_nudge_speaker_spherical` line 67–70) 와 정합.
- docstring `Raises` 절 (현 line 410–412) `ValueError` 항목 확장:
  ```
  ValueError
      When both frames are non-zero, when the resulting spherical distance
      is non-positive, or when the resulting spherical elevation is outside
      [-90, 90] degrees (non-physical — would flip the hemisphere).
  ```
