## Scope A — WFS CLI ergonomics — DONE at v0.7.0

### What landed

`roomestim/cli.py`:
- `_add_place_parser` and `_add_run_parser` each gain two flags
  (byte-symmetric):
  - `--wfs-f-max-hz FLOAT` (default 8000.0). Help: "Highest
    reproduction frequency for WFS spatial-aliasing bound (default
    8000.0)."
  - `--wfs-spacing-m FLOAT` (default None — derived). Help: "Override
    per-speaker spacing in metres (default: derived from
    --layout-radius and --n-speakers)."
- `_run_placement(...)` signature gains
  `wfs_f_max_hz: float = 8000.0` and
  `wfs_spacing_m: float | None = None` keyword args.
- The WFS branch derives `spacing_m` from `wfs_spacing_m` if supplied,
  else from `n_speakers` / `layout_radius` (existing formula).
- `place_wfs(...)` is wrapped in try/except `ValueError`. On the
  spatial-aliasing-bound failure path the CLI re-raises with a
  constructive remediation message naming both:
  - "max safe `--wfs-f-max-hz` for current spacing is X = c/(2*spacing_m) = ... Hz"
  - "minimum safe `--n-speakers` for current f_max_hz is Y = ceil(baseline_len/(c/(2*f_max))) + 1 = ..."
- `_cmd_place` and `_cmd_run` thread the new args via
  `getattr(args, "wfs_f_max_hz", 8000.0)` /
  `getattr(args, "wfs_spacing_m", None)` (defensive — `_cmd_ingest`
  and `_cmd_export` namespaces don't carry these flags).

The library-level `roomestim/place/wfs.py::place_wfs(...)` `ValueError`
API contract is **UNCHANGED**: `kErrWfsSpacingTooLarge: spacing_m=...
> c/(2*f_max)=...` still raises with the same message. The constructive
remediation message is built only at the CLI layer because the library
does not know `n_speakers` or `layout_radius` (which are CLI-derived);
building remediation strings inside the library would couple it to CLI
semantics. Keeping the wrap at the CLI layer matches the existing
layering.

### Math

Given baseline length `L` (= `2 * layout_radius`) and
`spacing = L / (n_speakers - 1)` (when `n_speakers > 1`):

- **Max safe `--wfs-f-max-hz` for current spacing**:
  `f_max_safe = c / (2 * spacing)` (Hz). This is the f_max that
  would make the bound `c/(2*f_max)` exactly equal the current
  spacing.
- **Minimum safe `--n-speakers` for current f_max_hz**:
  We need `spacing <= c/(2*f_max_hz) ≡ bound`, i.e.
  `(n - 1) >= L / bound`, so `n >= ceil(L/bound) + 1`.
  Verified empirically with `L=4.0, f_max=8000, c=343`:
  `bound = 343/16000 = 0.0214 m`;
  `L/bound = 4.0/0.0214 = 186.59`;
  `min_n = ceil(186.59) + 1 = 187 + 1 = 188`.
  Check: at n=188, spacing = 4.0/187 = 0.02139 < 0.02143 m ✓.

The smoke run on the v0.6.0 default invocation confirms both numbers
agree with the live CLI message:
`max safe --wfs-f-max-hz = 300.12 Hz` (= 343/(2*0.5714)) and
`minimum safe --n-speakers = 188`.

### Tests

`tests/test_cli_wfs_ergonomics.py` (+4 default-lane tests):
1. `test_run_wfs_default_n8_emits_constructive_error` — asserts
   stderr/stdout contains both `"max safe --wfs-f-max-hz"` and
   `"minimum safe --n-speakers"`.
2. `test_run_wfs_with_low_fmax_succeeds` — `--wfs-f-max-hz 300`
   unblocks default n=8 / radius=2.0 (300 Hz < 300.12 Hz max safe;

3. `test_run_wfs_with_explicit_spacing_succeeds` — `--n-speakers 200

4. `test_run_wfs_explicit_spacing_overrides_derived` — explicit

All 4 pass. Default-lane test count: 100 → 104.

## Scope C — Building_Lobby coupled-space ADR — DONE at v0.7.0

### Decision summary

Building_Lobby is excluded by default from any aggregate roomestim
metric (TASLP-MISC surface budget, perf doc MAE / median, future
octave-band aggregates). Per-room rows in those documents continue to
include Building_Lobby for transparency, but with the annotation
`(excluded — coupled space; see ADR 0014)`. The `_FURNITURE_BY_ROOM`
Building_Lobby exclusion (v0.6, ADR 0013 §Alternatives considered (d))
is now explicitly motivated by ADR 0014.

### Drivers

- TASLP §II-C: Building_Lobby is "irregular shape with coupled spaces"
  including a café, stairwell and staircase. Recording was made in a
  corner; `ACE_ROOM_GEOMETRY` 72.9 m³ shoebox describes that corner,
  not the full lobby.
- Sabine and Eyring assume diffuse field + simple bounded geometry;
  both fail on coupled spaces (Vorländer 2020 §4.4 — sound energy
  migrates between sub-volumes with different decay constants;
  multi-slope decay curves cannot be represented by single-T60
  estimators).
- Empirical: v0.6 perf doc Building_Lobby Sabine 500 Hz error
  `+1.425 s` (predicted 2.022 s vs measured 0.597 s; 3.4×
  over-prediction) is consistent with treating only the smaller-V
  coupled corner as the whole room. The 3.4× ratio is far outside
  the ±20% / ±50% warning bands used elsewhere in the audit ledger.

### Reverse-trigger conditions (per ADR 0014 §Reverse if)

ADR 0014 reverses if any of the following surface:
- A future ADR provides a coupled-space predictor (e.g.,
  Cremer/Müller two-room formula) AND per-sub-volume geometry for
  Building_Lobby.
- A non-canonical evidence channel (lab visit / author email) yields
  a single bounded effective volume that makes the shoebox model
  defensible for the recording corner.
- An external user contributes a coupled-space predictor and
  demonstrates > 50% improvement on Building_Lobby Sabine 500 Hz
  error without regressing the other 6 rooms.
