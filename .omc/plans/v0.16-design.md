## 1. 목표 — 사용자 4-제안 ①(재질만)/③(2D)/④(engine toggle) land

v0.15.2까지 RoomModel은 frozen + immutable이며 phone-scan 이후 material 정정 경로가 없었다 (사용자는 입력 YAML을 직접 편집해야 했음). v0.16에서 (a) `evolve_room` / `evolve_surface` helpers + (b) Surface Override Tab + (c) 2D blueprint export + (d) engine 검증 toggle 4개 surface를 동시 land한다. RoomModel frozen invariant는 유지 (모든 mutation은 `dataclasses.replace` chain으로 새 instance 반환).

### Scope IN

| # | Item | Surface |
|---|---|---|
| ① | 재질 override UI (드롭다운) | Web: Material Override Tab + Apply 버튼; Core: `evolve_room` / `evolve_surface` helpers |
| ③ | 2D 블루프린트 PNG/SVG export | Web: Blueprint Tab + Setup PDF 페이지; Core: `roomestim/viz/blueprint.py` NEW (top-down view) |

- Speaker nudge (v0.18 — Plotly 클릭 핸들러 + 좌표 입력 + 재 placement, OQ-32 NEW deferral).

## 2. 결정 (신규 D 3건 + 신규 ADR 3건; OQ 1건 신규 deferral)

### D-decisions

- **D39 NEW — RoomModel evolve helper API placement (`roomestim/edit.py` 신규 모듈)**
  - 선택 근거: `roomestim/model.py`는 frozen dataclass + 상수 (MaterialLabel enum + Absorption table)만 담는 "shape-only" 모듈. evolve 로직 (material change → absorption table lookup 사이드 이펙트)이 들어가면 model.py의 layering 단순성이 깨짐.
  - vs (A) `roomestim/model.py`에 inline: model.py가 두 책임 (shape + transition 로직) → reject. vs (B) `roomestim/reconstruct/evolve.py`: reconstruct는 RT60/ISM 예측 책임이지 RoomModel 변환이 아님 → reject. vs (C) `roomestim/edit.py` NEW top-level: shape transition (=edit) 책임 명확, 향후 v0.17 `add_obstacle()` / `remove_speaker()` 등 같은 모듈에 자연 누적 → **채택**.
  - 공개 시그니처:
    - `evolve_room(room: RoomModel, *, surfaces: list[Surface] | None = None, listener_area: ListenerArea | None = None, name: str | None = None) -> RoomModel` — `dataclasses.replace` wrapping + `canonicalize_ccw` 재호출 + `assert_finite` 재검증.
    - `evolve_surface(surf: Surface, *, material: MaterialLabel | None = None, polygon: list[Point3] | None = None) -> Surface` — material 변경 시 `absorption_500hz` / `absorption_bands`를 `MaterialAbsorption[material]` / `MaterialAbsorptionBands[material]`에서 auto-lookup.
    - `evolve_room_material(room: RoomModel, surface_index: int, material: MaterialLabel) -> RoomModel` — 1-surface convenience (web UI 직접 호출).
  - 역방향 기준: 외부 fork가 `evolve_room`을 dict-mutation으로 우회한다면 `from roomestim.edit import evolve_room` deprecation warning 발동 (v0.17+) — 이번 release는 default OFF.

- **D40 NEW — 음향 recompute trigger: manual Apply 버튼 (auto-debounce 거부)**
  - 선택 근거: AcousticReport 재계산은 ISM cascade (max_order=50 image source) → lab_room 1.9s 컴퓨트 — auto-debounce 200ms는 dropdown 빠른 연속 변경 시 worker 큐 폭주. 사용자 의도는 "여러 surface를 일괄 정정 후 한 번에 확인". manual button = 명시적 commit point + Plotly re-render 1회 보장.
  - vs (A) auto-debounce 200ms: ISM 1.9s × 5 surface 빠른 변경 → 9.5s worker 큐 폭주 → reject. vs (B) auto-submit on dropdown.change: 동일 큐 폭주 → reject. vs (C) Apply 버튼 + 변경 사항 미적용 상태 시각화 (dropdown label에 "*" 표시): UX 명확 + 큐 안전 → **채택**.
  - 부수 효과: Apply 버튼 click → `evolve_room_material` chain → `compute_acoustic_report` 재호출 → Plotly bar chart + JSON 직렬화 재생성. 3D viewer mesh는 불변 (surface 색만 갱신 — v0.16.x patch 후속).

- **D41 NEW — Blueprint 좌표계: y-down screen convention (architectural drawing 표준)**
  - 선택 근거: matplotlib default = y-up (math convention), 건축 도면 표준 = y-down (북쪽이 위, 종이 좌표). RoomModel 내부 좌표는 right-handed (x=right, y=up, z=forward per `Point3` docstring). top-down view에서 (x, z) 평면 투영하되 z축을 화면 위쪽으로 표시 = y-down screen ⇔ z-up world.
  - vs (A) y-up screen (matplotlib default): 익숙하지만 건축 도면 관례 위반 → reject. vs (B) y-down screen + `ax.invert_yaxis()` 후 z를 y축에 매핑: 건축 도면 관례 준수 + RoomModel 좌표계 보존 → **채택**. vs (C) 절대 (lat, lon) 매핑: 룸 회전 정보 부재 (phone-scan 출력은 룸 로컬 좌표) → reject.
  - 좌표 변환 명세: blueprint x축 = RoomModel x (right, m), blueprint y축 = RoomModel z (forward, m), `ax.invert_yaxis()` 호출하지 않음 — 대신 RoomModel `+z` (forward)가 화면 위쪽이 되도록 `ax.set_ylabel("z (forward, m, north-up)")` + extent 명시. 결과: 사용자가 룸 입구 (z=0)에 서서 본 뷰 = 화면 아래쪽이 입구. ADR 0032 §Coordinate 절에 결정 명문화.

- **D42 NEW — Engine 검증 precedence: CLI flag > ENV var > default ON (backward-compat)**
  - 선택 근거: v0.15.2까지 `roomestim export`는 `SPATIAL_ENGINE_REPO_DIR` env 또는 hardcoded `/home/seung/mmhoa/spatial_engine` 경로에서 schema를 읽어 강제 검증. v0.16에서 standalone YAML use-case (engine repo 없이 export만) 지원. 정직성 원칙: 사용자가 명시적으로 끄지 않는 한 검증 ON 유지 (silent skip 금지).
  - vs (A) default OFF (사용자가 켜야 함): silent ship 위험 — engine 호환성 회귀 무 발견 → reject. vs (B) default ON + CLI > ENV > default precedence: backward-compat + opt-out 명시 → **채택**. vs (C) ENV > CLI: 환경 변수가 명시적 flag를 override는 UX 안티패턴 → reject.
  - Web 사이드바 default: 체크박스 unchecked (=engine validation ON), 사용자가 명시적으로 체크하면 standalone 모드 (UI 라벨 "Standalone YAML (skip engine schema check)" — 부정 표현 명시).
  - CLI flag 형태: `--validate-engine PATH` (PATH = repo dir, ENV override) / `--no-engine-validation` (mutually exclusive group; default = ON with ENV/hardcoded fallback).

- **D43 NEW — Acoustic recompute on material change MUST 재실행 ISM/Eyring cascade (ADR 0009 invariant 보존)**
  - 선택 근거: material 변경 → `_band_alpha` 변경 → `_shoebox_per_band_alphas` 결과 변경 → ISM rt60 + Eyring rt60 모두 변경. ADR 0009 invariant (`ism_rt60 ≥ eyring_rt60 - 1e-6`)는 모든 evolved room에서도 유지되어야 함 — 그렇지 않으면 사용자가 임의 material 조합으로 invariant 위반 가능.
  - 결과: `tests/test_evolve_room.py`에 material-shuffle invariant 회귀 lock 추가 — 10 MaterialLabel × 6 surface 무작위 시드 5종 = 50 instance × invariant assert.

### ADR refs

- **ADR 0031 NEW — Material override 정책 + 음향 recompute trigger** (~120줄)
  - §Context: phone-scan 출력 RoomModel은 휴리스틱 material 추정 (e.g., RoomPlan은 wall material을 모름 → 전부 WALL_PAINTED). 사용자 정정 경로가 v0.15.2까지 부재.
  - §A Scope: web UI Material Override Tab + core `evolve_room` / `evolve_surface` / `evolve_room_material` 3 helpers. material 변경 → absorption table auto-lookup (사용자 임의 α 입력 거부 — closed enum 원칙 ADR 0002 + ADR 0008 보존).
  - §B Trigger: D40 — manual Apply 버튼.
  - §C Invariant: D43 — evolved room에서도 ADR 0009 (`ism_rt60 ≥ eyring_rt60 - 1e-6`) 유지.
  - §D RoomModel frozen 유지: 모든 mutation은 새 instance.
  - §E Reverse-criterion: 사용자 임의 α 입력 요구 ≥ 3건 또는 production 사용자가 "material lookup table이 부정확하다" 보고 시 v0.17+ 에서 ADR 0008 envelope 확장 + Surface에 `absorption_500hz_override: float | None` 추가.
  - §References: ADR 0002 (RoomModel shape), ADR 0008 (octave-band), ADR 0009 (ISM ≥ Eyring), ADR 0019 (MELAMINE_FOAM enum precedent for envelope 추가 cadence), D39 / D40 / D43.

- **ADR 0032 NEW — 2D 블루프린트 export 정책** (~100줄)
  - §Context: Setup PDF은 v0.13에서 3D viewer screenshot만 포함. 건축가 / 시공자 워크플로우에는 2D top-down + 치수 + 스피커 좌표 표가 필요.
  - §A Scope: PNG (300dpi raster) + SVG (vector, scalable for CAD import).
  - §B Coordinate: D41 — y-down screen ⇔ z-up world, x = right, z = forward (북쪽 = 화면 위).
  - §C Content layers: (1) floor polygon outline (검정 1.5pt), (2) wall labels (W0/W1/.../floor/ceiling), (3) listener area (semi-transparent green fill α=0.3), (4) speaker positions (빨간 점 + L/R/C/LS/RS 채널 라벨), (5) 치수 화살표 (longest wall 양 끝 좌표 + length), (6) 북쪽 화살표 + 1m scale bar.
  - §D Determinism: matplotlib Agg backend + `Path.from_iterable` 정렬 고정 + font fallback (`DejaVu Sans`, ships with matplotlib) → PNG byte-equal 회귀 lock 가능 (smoke test in §Phase 5).
  - §E Reverse-criterion: 사용자가 "건축가 standard symbol (door swing arc, window double line)" 요구 시 v0.17+ ADR 0032 §Status-update에서 symbol library 추가 또는 ezdxf 의존성 도입 결정.
  - §References: ADR 0024 (web-demo separate package), ADR 0027 (mesh format generalisation), D41.

- **ADR 0033 NEW — Engine 검증 토글 정책** (~80줄)
  - §Context: v0.15.2까지 `roomestim export`는 engine repo 검증 강제. standalone YAML 사용자 (예: 다른 audio engine 연동 사전 평가) 차단.
  - §A Scope: CLI `--validate-engine [PATH]` / `--no-engine-validation` mutually exclusive; web 사이드바 체크박스; ENV `SPATIAL_ENGINE_REPO_DIR` 후방 호환.
  - §B Precedence: D42 — CLI > ENV > default ON.
  - §C Default: ON (backward-compat). 명시적 OFF 시 export YAML에 `# WARNING: schema validation skipped` 헤더 코멘트 자동 삽입 (감사 추적).
  - §D Failure mode: validation 실패 시 export 차단 + stderr에 validation error 전체 + exit code 2 (현행 유지). `--no-engine-validation` 시 validation 미실행 → schema mismatch silent ship 가능 (사용자 책임).
  - §E Reverse-criterion: 사용자가 "여러 engine target schema" 요구 시 v0.18+ 에서 `--validate-engine PATH:target_name` syntax 도입.
  - §References: ADR 0024, ADR 0027, D42, OQ-31 (engine schema 다중 target deferral).

- **ADR 0030 §Status-update-v0.16 NEW** (~25줄) — D22 audit-trail-discipline 패턴.
  - Item I — Material Override UI land (D39 + D40 + D43 + ADR 0031).
  - Item J — 2D Blueprint export land (D41 + ADR 0032).
  - Item K — Engine 검증 토글 land (D42 + ADR 0033).
  - 정직성 노트: v0.16 land 후에도 "phone-scan accuracy" 메인 갭은 미해결 (이는 RoomPlan / Polycam 측 한계). v0.16은 "scan 후 정정 경로" 만 제공.
