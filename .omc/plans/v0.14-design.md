### §0.1 Why-this-scope-not-alternatives

most-likely-alternative scope shapes that the v0.14 planner round
considered and rejected.

**(a) "Split v0.14 into v0.14 (Item A admin hard-wall closure ONLY) +
v0.15 (Item B + C ISM bundle DELIBERATE)"** — REJECTED.
Both hard-wall closure (D27 / Item A) and ISM bundle (OQ-15 / Item B)
are scheduled to land at v0.14 by INDEPENDENT cadences (D27 schedule:
v0.11 NEW → v0.12 1st defer → v0.13 2nd defer → v0.14 hard wall; OQ-15

  _SPEED_OF_SOUND_MPS: float = 343.0
  _DEFAULT_MAX_ORDER: int = 50
  _DEFAULT_ENERGY_THRESHOLD_DB: float = -60.0

  def image_source_rt60(
      volume_m3: float,
      dimensions_m: tuple[float, float, float],
      surface_areas: dict[MaterialLabel, float],
      *,
      max_order: int = _DEFAULT_MAX_ORDER,
      energy_threshold_db: float = _DEFAULT_ENERGY_THRESHOLD_DB,
  ) -> float:
      """Shoebox-only image-source RT60 estimate at 500 Hz mid-band.

      For shoebox L×W×H with per-wall α coefficients, integrates the

      at order (nx, ny, nz) has total path length scaled by L/W/H; cumulative

  Implementation sketch: for shoebox with walls of materials (wall_x_pos,
  wall_x_neg, wall_y_pos, wall_y_neg, floor, ceiling) — each material has

  but for shoebox dimensions_m provides authoritative L/W/H. Image
  positions at integer-lattice points (nx*L, ny*W, nz*H); for each
  position compute path length r = sqrt(...); cumulative reflection
  attenuation = (1 - α_x)^|nx| * (1 - α_y)^|ny| * (1 - α_z)^|nz|;
  energy contribution = atten^2 / r^2 (1/r^2 spreading). Sum energy
  over all images with order |nx| + |ny| + |nz| ≤ max_order; track
  energy decay as function of t = r / c; fit linear regression on
  log(energy) vs t; RT60 = -60 / slope.

  point-receiver in the centre); v0.15+ may add per-receiver position
  ensembling (averaging across listener positions). For RT60
  characterisation (vs. early-reflection prediction) the centre-pair

## §4 Pre-mortem (DELIBERATE-mode; 3 scenarios)

Per ralplan DELIBERATE convention, three failure scenarios examined
before execution:

**Scenario 1 (LIKELY): Item B analytic shoebox validation fails at
moderate absorption (α=0.50) regime.**
- *Setup*: Item B implementation is correct in the low-absorption limit
  (where ISM converges to Sabine) but moderate-absorption regime exposes
  an absorption-bookkeeping bug — e.g., reflection-order energy attenuation
  uses `(1 - α)` per-bounce but the implementation accidentally computes
  energy (not amplitude) attenuation as `(1 - α)^2` per-bounce. Cube room
  at α=0.50 produces ISM RT60 = 0.25 s vs Eyring 0.40 s (delta > ±5%).
- *Detection*: §0.4 STOP rule #2 fires at first test run.
- *Mitigation*: Escalate to planner. Acceptable executor action: inspect
  per-bounce attenuation; verify amplitude vs energy convention against
  Vorländer 2020 §11.2 (energy attenuation factor is `(1 - α)` per
  bounce; amplitude is `sqrt(1 - α)`); fix. Re-run validation. If
  still fails after parameter tuning AND convention check, escalate
  to planner for v0.14.1 patch or v0.14 → v0.13.1 downgrade.
- *Impact if mitigation fails*: v0.14 ships with Item A only (path γ
  hard-wall closure); Item B re-deferred to v0.14.1 patch or v0.15.0.

**Scenario 2 (MEDIUM-LIKELY): External acquisition succeeds mid-cycle
(Vorländer verbatim arrives via SNU library ILL OR user provides
extracted value).**
- *Setup*: §0.4 STOP rule #7 fires. Executor must switch from path γ
  (planner-locked default) to path α (verbatim closure).
- *Detection*: STOP-and-consult per rule.
- *Mitigation*: HAPPY PATH. ADR 0022 §Decision body records the
  closure path actually taken (α with verbatim value); ADR 0019
  §Status-update-2026-05-NN records HARD-WALL CLOSURE under α; §Drivers
  honesty-leak entry under γ is REDACTED in favour of α closure record.
  Library row update IF verbatim value differs from 0.85 (inside
  [0.80, 0.95] envelope). Lab A11 re-run. Test count delta ~0
  (renames; no new tests).
- *Impact if mitigation succeeds*: v0.14 ships under path α (or β); D27
  hard wall closed under verbatim acquisition; honesty discipline +
  audit-trail strengthened.

**Scenario 3 (LOW-LIKELY): Item D reverse fires (Office_1 ratio > 1.15
AND user requests in-release predictor-default switch).**
