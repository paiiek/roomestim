## 1. 목표 — 사용자 4-제안 ② (출력 편집)

v0.17.0까지 roomestim의 데이터 흐름은 **단방향 read-only** 였다: phone-scan →
RoomModel (편집 가능: 재질/객체) → placement 알고리즘 → `PlacementResult` →
`layout.yaml` write. 일단 `layout.yaml` 이 쓰여지면 그 안의 스피커 좌표를
**다시 읽어 미세 조정한 뒤 되쓰는** 경로가 없었다. 사용자가 자동 배치 결과
(VBAP ring 등) 를 받아본 뒤 "3번 스피커를 5° 오른쪽으로", "전체 반경을 0.2m
줄여서" 같은 수동 조정을 하려면 YAML을 손으로 편집해야 했고, 그 결과가 engine
schema (`geometry_schema.json`) 를 여전히 만족하는지 (min_speaker_count, finite
sweep, az/x 상호배타) 재검증할 수단이 없었다.

v0.18에서 (a) `roomestim/edit.py` 에 `evolve_placement(...)` (frozen-respecting,
`evolve_room` 패턴 미러) + `nudge_speaker(...)` convenience helper, (b)
**기존** reader/writer (`roomestim/io/placement_yaml_reader.py` +
`roomestim/export/layout_yaml.py`) 를 재사용·보강하여 round-trip 충실도를
명시적 계약으로 정식화 (parse → edit → write → re-validate), (c) Web 3D 뷰어의
스피커 클릭 선택 + nudge 패널 (az/el/dist 또는 Δ 입력) → `evolve_placement` →
재렌더 + engine 재검증, (d) CLI `roomestim edit` 신규 subcommand (layout.yaml
편집 + diff 출력 + 재검증) 4 surface를 land한다.

**핵심 grounding 사실 (이미 검증, 스켈레톤 오류 정정)**: 스켈레톤이 가정한
`roomestim/load_layout.py` NEW 는 **틀렸다**. 파싱(`read_placement_yaml`)과
직렬화(`write_layout_yaml`)는 **이미 존재**한다. v0.18 round-trip은 신규 모듈이
아니라 *기존 reader + 기존 writer + 신규 evolve helper* 로 달성된다 (D48).
또한 `PlacedSpeaker` 는 스켈레톤이 가정한 `az/el/radius` 필드가 아니라
**`channel: int`, `position: Point3` (Cartesian), `aim_direction: Point3 | None`,
`notes: str`** 필드를 가진다 (frozen dataclass). az/el/dist는 직렬화 *형태* 일
뿐 모델 내부 표현이 아니다 — nudge API는 Cartesian과 spherical 양쪽 입력을
지원하되 내부는 항상 `Point3` 로 정규화한다 (D49).

**round-trip 충실도의 진실 (planner가 직접 코드 실행으로 확인)**: 현재
`read_placement_yaml` 은 (i) `aim_direction` 을 **버린다** (writer는
`x_aim_az_deg`/`x_aim_el_deg` 를 emit하지만 reader는 무시 → 항상 `None`),
(ii) `notes` 를 **버린다** (직렬화 자체 안 됨), (iii) **comment 미보존** (`#
WARNING` 헤더는 재생성), (iv) **`target_algorithm` 알고리즘 라벨을 round-trip
하지 않는다** (reader `placement_yaml_reader.py:46-54` 는 `regularity_hint` +
`x_wfs_f_alias_hz` 존재 여부로 **WFS-vs-VBAP 만** 추론 → DBAP/AMBISONICS 는
"VBAP" 로 붕괴), (v) per-speaker `id` 를 **버린다** (writer는 `id == channel`
emit; reader는 `id` 미사용 — channel 만 보존). 그러나 (vi) `position` 은
**구조적으로 보존**됨 (az/el/dist → cartesian → az/el/dist 가 ~1e-9 안정),
(vii) `channel` / `regularity_hint` / `wfs_f_alias_hz` 보존. 따라서 스켈레톤의
"byte-equal when no edit" 주장은 **명시적 aim을 가진 layout에 대해 거짓**이고,
"`target_algorithm` 보존" 도 **DBAP/AMBISONICS 에 대해 거짓**이다 (D27/D35 정직성
규율 — 무조건 보장으로 두지 않음). v0.18은 round-trip을 정직하게 두 단계로
정식화한다 (D50):
- **Level 1 (position round-trip, v0.18 boundary, {VBAP, WFS} 만)**: position/
  channel/regularity 는 구조적 동치 (≤ 1e-9); `target_algorithm` 은 **{VBAP, WFS}
  에서만 보존** (reader 가 WFS 판별 가능 — `x_wfs_f_alias_hz` 키 존재 시 WFS,
  아니면 VBAP). **DBAP/AMBISONICS 는 read 시 "VBAP" 로 읽혀 알고리즘 라벨이
  round-trip 하지 않는다** (OQ-38 deferral — engine schema 에 `x_target_algorithm`
  extension 도입 시 해소). aim 은 v0.18에서 **명시적으로 복원** (reader가
  `x_aim_az_deg`/`x_aim_el_deg` 를 읽어 `aim_direction` 재구성). notes / per-speaker
  `id` 는 `layout.yaml` 에 충실 슬롯이 없으므로 (engine schema 미정의) **편집
  round-trip 대상에서 제외** + 명시적 한계로 문서화 (notes = OQ-37; id = writer가
  channel 로 재생성).
- **Level 2 (byte-equal, OUT OF SCOPE)**: comment 보존 + key 순서 + 부동소수
  포맷 완전 동일 = ruamel.yaml round-trip 모드 필요 → v0.18 비-목표 (D51).

### Scope OUT (v0.19+ 로 deferral)

- **`notes` 필드 round-trip** — `layout.yaml` engine schema (`geometry_schema.json`)
  에 per-speaker note 슬롯이 없음. extension key `x_notes` 도입은 engine 협의 필요
  → OQ-37 NEW deferral. v0.18 nudge는 position/aim만 편집; notes는 보존 불가
  (read시 `""`) 명시.
- **byte-equal round-trip (comment + key order + float 포맷 완전 보존)** — D51 비-목표.
  ruamel.yaml round-trip 모드 도입은 새 의존성 + `write_layout_yaml` 의 `yaml.safe_dump`
  교체 = 광범위 회귀 위험. v0.18은 Level 1 (구조 동치) 만.
- **`--schema 0.1` 다운그레이드 export flag** — v0.17 §Status-update에서 forward-ref된
  room.yaml 다운그레이드 정책. v0.18 scope (layout 편집) 와 직교 → OQ-36 NEW
  (이미 ADR 0030 §Status-update-v0.17 line 343 + ADR 0034 §B + decisions.md
  D44에서 forward-ref; v0.18에서 정식 allocate 후 deferral).
- **다중 스피커 동시 nudge / 전체 ring 스케일** — v0.18은 1 스피커 1 Δ 단위.
  "전체 반경 0.2m 축소" 같은 group transform은 `evolve_placement(speakers=[...전체
  재계산])` 로 가능하지만 전용 helper (`scale_layout(result, factor)`)는 v0.19+
  (사용자 요청 cadence 관찰).
- **layout.yaml 재배치 알고리즘 재실행** — 편집은 *기존 좌표의 미세 조정* 만.
  algorithm 자체 재실행 (VBAP → WFS 전환 등) 은 `roomestim place` 경로 (기존).

### 비-목표 (영구 거부)

- **`PlacedSpeaker` 를 mutable로 전환** — `frozen=True` 유지. 모든 nudge는
  `dataclasses.replace` chain (`evolve_placement`/`nudge_speaker` 경유). 사용자가
  `result.speakers[i].position = ...` 직접 mutation 요청 시 거부 (FrozenInstanceError).
- **`PlacementResult` 에 임의 metadata dict 추가** — `target_algorithm` /
  `regularity_hint` / `layout_version` / `wfs_f_alias_hz` 는 고정 스키마. 임의
  `x_*` extension은 `layout.yaml` write 단계 (engine schema `additionalProperties:
  true`) 에서만 처리하며 model에는 추가 안 함 (ADR 0036 §Reverse-criterion 발동
  조건: engine schema가 필수 metadata 추가 시).
- **두 좌표 표현(spherical Δ + Cartesian Δ) 동시 입력** — `nudge_speaker` 가
  ValueError. 단일 권위 frame 변환 (`roomestim/coords.py`) 우회 금지 (D49).
- **comment-preserving YAML writer를 core에 도입** — D51. core는 `yaml.safe_dump`
  단일 직렬화 권위 유지.

## 2. 결정 (신규 D 5건 + 신규 ADR 1건; OQ 3건 신규 deferral)

### D-decisions

- **D48 NEW — Round-trip은 신규 모듈(`load_layout.py`) 없이 기존 reader+writer+
  신규 evolve helper로 달성 (스켈레톤 `load_layout.py` 오류 정정)**
  - 선택 근거: `roomestim/io/placement_yaml_reader.py::read_placement_yaml` (parse)
    와 `roomestim/export/layout_yaml.py::write_layout_yaml` (serialize) 가 **이미
    존재**하며 양쪽 다 `__all__` 공개 + 테스트 커버됨 (`tests/test_export_layout_yaml.py`).
    스켈레톤의 `roomestim/load_layout.py` NEW 는 이 두 모듈을 재발명하는 중복 —
    D29 lane separation·DRY 위반. v0.18은 (i) reader에 aim 복원 보강 (②-3), (ii)
    `roomestim/edit.py` 에 `evolve_placement`/`nudge_speaker` 추가 (②-1/②-2) 만으로
    round-trip 완성: `read_placement_yaml(p)` → `nudge_speaker(r, ...)` →
    `write_layout_yaml(r2, p2)`.
  - vs (A) 신규 `roomestim/load_layout.py` (스켈레톤): 기존 reader/writer 중복
    → reject. vs (B) reader/writer를 `roomestim/layout.py` 단일 모듈로 통합
    리팩터: v0.18 scope 밖 대규모 이동 + 기존 import 경로 (cli.py, app.py)
    회귀 위험 → reject. vs (C) 기존 reader 보강 + edit.py 확장 → **채택**.
  - Scope: core (`roomestim/io/` + `roomestim/edit.py`). Reverse if: 향후 layout
    편집 로직이 reader/writer/edit 3곳에 분산되어 응집도 저하 시 `roomestim/layout/`
    package로 통합 리팩터 (v0.19+ 별도 결정).
  - ADR ref: ADR 0036 §A.

- **D49 — nudge 입력 단위: spherical Δ (az/el/dist 도·미터) XOR Cartesian Δ
  (미터); 동시 지정 거부; 내부 정규화는 `coords.py` 단일 권위**
  - 선택 근거: 사용자 mental model은 두 가지 — (i) "5° 오른쪽으로 / 0.2m 더
    멀리" (spherical, ring 배치 편집에 자연스러움), (ii) "x를 0.1m" (Cartesian,
    grid 배치). `PlacedSpeaker.position` 은 `Point3` (Cartesian) 단일 표현이므로
    spherical Δ는 `cartesian_to_pipeline` → (az,el,dist) → Δ 적용 →
    `yaml_speaker_to_cartesian` 로 다시 Cartesian. 두 모드 동시 지정은 의미
    모호 (어느 frame 먼저?) → ValueError. 모든 변환은 `roomestim/coords.py`
    (single sign-flip authority, 2026-03-01 inversion bug 방지 주석 참고) 경유 —
    nudge 모듈은 삼각함수 직접 호출 금지.
  - granularity: web UI는 az/el step **1°**, dist step **0.05m** (slider/number
    `step`); Cartesian step **0.05m**. CLI는 임의 float (사용자 책임). 스켈레톤의
    "1°? 5°? 0.1m?" 질문에 대한 답 = web default step 1° / 0.05m (사용자가 더 큰
    이동 시 여러 번 또는 number 직접 입력). 스냅(snap-to-grid)은 비-목표.
  - vs (A) Cartesian만: ring 편집 시 az 직관 상실 → reject. vs (B) spherical만:
    grid/비대칭 배치 어려움 → reject. vs (C) 둘 다 + 상호배타 → **채택**.
    vs (D) 둘 다 + 합성 (Cartesian 먼저 후 spherical): frame 순서 의존 +
    재현성 저하 → reject.
  - Scope: core (`roomestim/edit.py::nudge_speaker`). Reverse if: 사용자가 단일
    Apply에서 두 frame 합성 nudge 요청 ≥ 2건 → 명시적 순서 계약 (Cartesian-first)
    문서화 후 허용.
  - ADR ref: ADR 0036 §B.

- **D50 — round-trip 충실도 Level 1 (position+aim 구조 동치) 정식 계약; aim은
  v0.18에서 reader 복원으로 비손실화**
  - 선택 근거: planner 직접 실행 확인 — 현재 reader는 `aim_direction`/`notes`
    drop. position은 az/el/dist 직렬화 후에도 ≤1e-9 구조 보존. v0.18 nudge가
    aim을 편집·표시하려면 **읽기 시 aim 복원 필수** (안 그러면 nudge 후 aim이
    전부 origin-default로 리셋되어 사용자 의도 파괴). 따라서 reader가
    `x_aim_az_deg`/`x_aim_el_deg` 존재 시 `aim_direction` 복원 (dist=1.0 unit
    벡터; aim은 방향만 의미 — `cartesian_to_pipeline` 이 magnitude 폐기하므로
    write→read→write가 방향 보존). 키 부재 시 `None` (구 layout backward-compat).
  - Level 1 계약 ({VBAP, WFS} 에 한정): `read(write(r)) == r` 가 (position ≤1e-9,
    channel ==, regularity ==, wfs_f_alias_hz ≤1e-9, aim_direction 방향 ≤1e-9)
    에서 성립. **`target_algorithm` 은 {VBAP, WFS} 에서만 보존** — `target_algorithm
    ∈ {DBAP, AMBISONICS}` 인 result 는 read 시 "VBAP" 로 붕괴하므로 라벨 동치를
    **계약에서 명시적으로 제외** (OQ-38; reader `placement_yaml_reader.py:46-54`
    가 regularity + WFS 키만으로 추론). `notes` 와 per-speaker `id` 는 계약 제외
    (충실 슬롯 없음 — notes = OQ-37; id 는 writer 가 channel 로 재생성).
  - **idempotency 의 명시적 branch-switch 근거 (HIGH-1)**: writer 는 **항상**
    `x_aim_az_deg`/`x_aim_el_deg` 를 emit 한다 — `aim_direction is None` 일 때는
    toward-origin 기본값 (negation of position; `layout_yaml.py:109-112`),
    `aim_direction` 이 set 일 때는 명시 분기 (`layout_yaml.py:113-116`). 따라서
    **첫 write→read 후 모든 speaker 가 `aim_direction != None` 이 되고, 두 번째
    write 는 무조건 명시 분기 (`:113-116`) 를 탄다** (None 분기가 아님). 두 write
    가 byte-equal 인 이유: aim 은 방향만 의미하고 `cartesian_to_pipeline` 의
    `az = atan2(x, z)` / `el = atan2(y, √(x²+z²))` 는 **scale-invariant** (반지름
    1.0 unit 벡터로 복원해도 atan2 비율이 동일) — `coords.py` 단일 권위. 즉
    write→read→write 는 두 번째 write 부터 안정 고정점 (Gate 20 — explicit-aim).
  - **default-aim (never-authored, toward-origin) layout 도 동일 고정점**: 한 번도
    aim 을 명시한 적 없는 layout (writer 가 toward-origin 기본값으로 emit) 을
    read 하면 reader 가 그 toward-origin 방향을 `aim_direction` 으로 복원 →
    재-write 시 명시 분기로 동일 az/el emit → byte-equal. 이 경로는 **별도
    수용 게이트** (Gate 21 — default-aim 고정점) 로 회귀-lock.
  - vs (A) aim drop 유지 + nudge가 aim 무시: nudge 후 모든 스피커가 origin-aim
    리셋 → 사용자 데이터 파괴 → reject. vs (B-opt) reader 복원을 opt-in

    안티테제 결론 참고) → reject. vs (B) aim 무조건 복원 (Level 1, default) →
    **채택** (export 버그도 함께 수리; HIGH-2 export byte-gate 가 넓어진 blast

    def evolve_placement(
        result: PlacementResult,
        *,
        speakers: list[PlacedSpeaker] | None = None,
        regularity_hint: str | None = None,
        layout_name: str | None = None,
    ) -> PlacementResult:
        """Return a new PlacementResult with given fields replaced.

        Mirrors :func:`evolve_room`: input never mutated, speakers list is
        shallow-copied, every PlacedSpeaker is frozen. Finite-validates each
        speaker position (and aim, if present).
        """
        new_speakers = list(speakers) if speakers is not None else list(result.speakers)
        for i, sp in enumerate(new_speakers):
            assert_finite(sp.position.x, field=f"speakers[{i}].position.x")
            assert_finite(sp.position.y, field=f"speakers[{i}].position.y")
            assert_finite(sp.position.z, field=f"speakers[{i}].position.z")
            if sp.aim_direction is not None:
                assert_finite(sp.aim_direction.x, field=f"speakers[{i}].aim.x")
                assert_finite(sp.aim_direction.y, field=f"speakers[{i}].aim.y")
                assert_finite(sp.aim_direction.z, field=f"speakers[{i}].aim.z")
        return replace(
            result,
            speakers=new_speakers,
            regularity_hint=regularity_hint if regularity_hint is not None else result.regularity_hint,
            layout_name=layout_name if layout_name is not None else result.layout_name,
        )

        Mixing the two frames raises ValueError. All frame conversion goes
        through roomestim.coords (single sign-flip authority).
        """
        n = len(result.speakers)
        if not (0 <= speaker_index < n):
            raise IndexError(f"speaker_index={speaker_index} out of valid range [0, {n})")
        spherical = (daz_deg, del_deg, ddist_m) != (0.0, 0.0, 0.0)
        cartesian = (dx, dy, dz) != (0.0, 0.0, 0.0)
        if spherical and cartesian:
            raise ValueError("nudge_speaker: spherical Δ and Cartesian Δ are mutually exclusive")
        sp = result.speakers[speaker_index]
        p = sp.position
        if cartesian:
            new_pos = Point3(p.x + dx, p.y + dy, p.z + dz)
        else:
            az_rad, el_rad, dist = cartesian_to_pipeline(p.x, p.y, p.z)
            az2 = math.degrees(az_rad) + daz_deg
            el2 = math.degrees(el_rad) + del_deg
            d2 = dist + ddist_m
            if d2 <= 0.0:
                raise ValueError(f"nudge_speaker: resulting dist {d2} must be > 0")
            nx, ny, nz = yaml_speaker_to_cartesian(az2, el2, d2)
            new_pos = Point3(nx, ny, nz)
        new_sp = replace(sp, position=new_pos)
        new_speakers = list(result.speakers)
        new_speakers[speaker_index] = new_sp
        return evolve_placement(result, speakers=new_speakers)
    ```

    def validate_placement(
        result: PlacementResult, *, schema_path_override: str | None = None
    ) -> list[str]:
        """Collect validation issues WITHOUT raising or writing a file.

        Independent re-check of R10 (min_speaker_count) + R11 (finite sweep) +
        engine Draft202012 schema. Does NOT call write_layout_yaml and does NOT
        alter the writer's raise path — the writer keeps raising for safety;
        this collector exists only for CLI/web UX. ([] = valid.)
        """
        errors: list[str] = []
        try:
            min_n = _min_speaker_count(result.regularity_hint)
            if len(result.speakers) < min_n:
                errors.append(f"{kErrTooFewSpeakers}: needs >= {min_n}, got {len(result.speakers)}")
        except ValueError as e:
            errors.append(str(e))
        try:
            data = placement_to_dict(result)
            _sweep_finite(data, path="")
            schema_file = _resolve_schema_file(schema_path_override)
            with schema_file.open("r", encoding="utf-8") as fh:
                schema = json.load(fh)
            Draft202012Validator(schema).validate(data)
        except Exception as e:  # ValidationError, ValueError, OSError → string for UX only
            errors.append(str(e))
        return errors
    ```
