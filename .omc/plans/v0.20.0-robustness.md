## Context

`roomestim/export/layout_yaml.py:57-59` carries a hardcoded ABSOLUTE fallback
`/home/seung/mmhoa/spatial_engine/proto/geometry_schema.json`. Resolution today
(`_engine_schema_path`, l.62-69):

```
SPATIAL_ENGINE_REPO_DIR env (only if <dir>/proto/geometry_schema.json exists)
  → else _DEFAULT_ENGINE_SCHEMA_PATH (the hardcoded absolute path)
```

`--validate-engine PATH` (CLI / `schema_path_override`) takes precedence over both via
`_resolve_schema_file` (l.79-90). **Verified**: the hardcoded path EXISTS on disk right
now (`-rw-rw-r-- 2692 bytes`), so the green suite is an accident of this machine — on any
other host with no env var set and no `--validate-engine`, default-ON validation would
fail with a bare `FileNotFoundError` deep inside `schema_file.open()`.

ADR 0033 §B/§E already own the policy: precedence is `CLI > ENV > hardcoded default`, and
§E reverse-criterion says *"when the hardcoded default becomes permanently unavailable →
remove default + make ENV var required."* The path is not yet unavailable, so this cycle
honors §E intent without the breaking removal: keep the default, but replace the silent,
non-portable failure with a **descriptive, actionable error** when nothing resolves.

## Design Decisions (decisive)

### ④ — Recommendation: (a) descriptive error, default RETAINED as documented constant

Of the three options:
- **(a) descriptive error** when neither env-var nor override resolves a file. ✅ CHOSEN.
- (b) optional/warn-and-skip: REJECTED — collides with ADR 0033 §C/§D which already give
  the user an explicit `--no-engine-validation` opt-out with an audit header. A *silent*
  warn-skip would re-introduce the dishonesty §C was written to prevent.
- (c) vendor a pinned copy: REJECTED — the module docstring (l.3 "read at write time,
  never copied") and ADR 0027/0033 deliberately keep the engine schema un-vendored to
  avoid drift; D64 reaffirms mesh output is never validated against a vendored schema.

**Concrete shape**: keep `_DEFAULT_ENGINE_SCHEMA_PATH` as the documented fallback (per
ADR 0033 §B — do NOT break the documented `CLI > ENV > default` chain), but harden
`_engine_schema_path()` / the open sites so that when the *finally resolved* path does
not exist, a single descriptive error is raised naming both escape hatches:

```
raise FileNotFoundError(
    f"{kErrEngineSchemaNotFound}: engine geometry schema not found at {path}. "
    f"Set SPATIAL_ENGINE_REPO_DIR=<spatial_engine repo dir>, pass "
    f"--validate-engine <dir>, or use --no-engine-validation to skip "
    f"(ADR 0033)."
)
```

This is a behavior change ONLY on hosts where the schema is genuinely absent (bare
`FileNotFoundError` → actionable message). On this machine and any machine with the env
var or repo present, behavior and byte output are unchanged → suite stays green, RT60
byte-equal preserved. New sentinel `kErrEngineSchemaNotFound` mirrors the existing
`kErr*` convention in `model.py`.

### Add-on #1 — Recommendation: IN

`tests/fixtures/lab_room_vertex_color.ply` was verified to have **12 faces** (closed box),
so the current `vertices.shape[1] != 3` guard is not bypassed by it. A genuine points-only
PLY IS reachable: `trimesh.load(force="mesh")` returns a `Trimesh` with `len(faces)==0`,
which currently slips through to convex-hull logic (undefined downstream). Add a guard
right after the vertex-shape check (`mesh.py:~82`). Requires a tiny new 0-face fixture.
Small, self-contained, closes OQ-21.

### Add-on #2 — Recommendation: IN (remove export)

`place_ambisonics` (`place/ambisonics.py`) is a pure `NotImplementedError` stub. Verified
consumers: ONLY `place/__init__.py:6,17` (re-export). NOT in `dispatch.py`, NOT in CLI
`--algorithm {vbap,dbap,wfs}` choices, NOT imported by tests (the one ambisonics test,
`test_layout_round_trip.py:162`, exercises `TargetAlgorithm` collapse, not this function).
Remove the import + `__all__` entry from `place/__init__.py` and delete `ambisonics.py`.
Leaves zero misleading public surface. (If a future v0.x revives ambisonics it gets a real
impl + ADR; the stub adds no value today.)

## Version bump — MINOR v0.20.0 (SemVer justification)

The project was critiqued for version inflation, so this is justified narrowly: ④ changes
the **observable behavior** of engine-schema resolution for callers without the engine repo
(bare `FileNotFoundError` → guided actionable error; the default-ON contract surface changes).
Add-on #1 adds a new validation error path for a previously-undefined input. These are
additive/behavioral changes to a public contract → MINOR, not a pure-internal PATCH.
(A docstring-only or pure-refactor cycle would be PATCH; this is not that.) No removals that
break documented public API in a breaking way (ambisonics stub was never functional and never
in CLI/dispatch). **roomestim_web is NOT touched → version stays `0.16-web.0`.**

## Risk / Rollback

- **Risk (low)**: a CI/dev host that currently relies on the silent hardcoded fallback (env
  unset, repo at the canonical absolute path) is unaffected — the path still resolves. Only
  hosts where it is genuinely absent see the new (better) error. Mitigation: error message
  names all three escape hatches.
- **Risk (low)**: ambisonics removal could break an undiscovered external importer. Mitigation:
  grep confirmed zero internal consumers beyond the re-export; documented in D67 + RELEASE_NOTES.
- **Rollback**: all three changes are independent commits; revert any one without affecting
  the others. ④ is a single-source guard; mesh guard is one `if`; ambisonics is a deletion.
