## 2. Phase별 구현

### Phase 1 — OQ-27 SHA-256 pin
- `scripts/fetch_web_data.py`: `KEMAR_SOFA_SHA256` + `LIBRIVOX_MP3_SHA256` 상수 신규.
- `fetch_kemar(...)`: `_download_file(..., expected_sha256=KEMAR_SOFA_SHA256)`.
- `fetch_librivox(...)`: 동일 처리 (MP3 단계에서 SHA 검증).
- `_download_file` WARNING-only 분기 (line 86-89) 유지 (테스트가 None 전달 케이스 검증).
- ADR 0029 §A §Status-update-v0.12-web.5 NEW: "SHA pin landed".
- OQ-27 closure 문구.

### Phase 2 — MINOR-1 wire-up
- `app.py:_on_submit` 시그니처는 그대로 (7 입력), 반환을 6-tuple → 7-tuple로 확장: `(viewer, report_chart, report_json, pdf, binaural, binaural_status_md, raw)`.
- `gr.Markdown` 컴포넌트에 변수 `binaural_status_md` 부여.
- `submit_btn.click(outputs=[..., binaural_audio, binaural_status_md, raw_file])` 7요소.
