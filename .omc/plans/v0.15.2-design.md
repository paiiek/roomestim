## 1. 목표 — v0.15.1 known-gap closure + D29 lane scope 명시화

v0.15.1에서 `roomestim/geom/polygon.py` shared util은 core 내부에 land했으나
`roomestim_web/report.py`는 자체 `_polygon_area_3d` / `_shoelace_2d` /
`_room_volume` 3 함수 duplicate를 유지함 (web 디렉토리 byte-equal 목적 +
D29 lane separation 본래 의미 보존). v0.15.1 code-review LOW retention 노트는
"core 내부 추출만 land, web→core import는 별도 ADR 결정 후 v0.15.2 land" 약속.
v0.15.2에서 이 약속을 닫는다. 동시에 v0.15.1 LOW-2 (`ace_challenge.py`에서
제거된 `_room_volume`에 대한 deprecation shim 부재)에 대한 closure 결정을
명문화한다 (clean-close, 근거는 §2-2 + §3 Phase 1).

## 2. 결정 (D29 amendment 1건, 신규 ADR 없음, 신규 OQ 없음)

- **ADR 0029 §Cross-lane-geom-amendment NEW** — D29 lane separation은 본래
  "web 의존성이 core 디렉토리에 새지 않게 한다" (단방향). 역방향 (web → core
  내부 stable public util import)은 lane separation을 침범하지 않으며 오히려
  duplicate-elimination 측면에서 권장된다. v0.15.2부터 `roomestim_web/**`는
  `roomestim.geom.*` public 심볼(`polygon_area_3d`, `room_volume`,
  `shoelace_2d`) import 가능. 역방향(core → web) 금지는 불변. **선택 근거**:
  (A) ADR 0029 §Cross-lane-geom-amendment in-place 신규 절 vs (B) ADR 0031
  NEW vs (C) duplicate 영구 유지. (A) 채택 — D29 의 lane scope를 좁히는
  factual refinement (방향성 명문화)는 ADR 0029의 기존 §B/§C 단방향 데이터
  흐름 원칙의 직접 연속이므로 D28-P1 hybrid 패턴 (§Status-update 블록)에
  적합. (B)는 새 결정 사항(예: 새 invariant, 새 호출 그래프) 없이 ADR 번호
  증가시키는 noise. (C)는 v0.15.1 code-review retention 노트 약속 위반 +
  duplicate KLOC 증가. D26 forbidden-indefinite-deferral 적용 안 함 (v0.15.1
  retention은 한 cycle 안에서 land 약속).
- **ADR 0030 §Status-update-v0.15.2 NEW** — Item G (web geom dedup land) +
  Item H (LOW-2 clean-close) closure 기록. D22 audit-trail-discipline 패턴 (v0.10.1 / v0.15.1 precedent).
- **LOW-2 clean-close 근거 명문화** — `_room_volume`은 underscore prefix +
  `ace_challenge.py` `__all__` 미정의 (default = 모든 top-level symbol, 단
  underscore prefix는 wildcard import 제외) + 외부 consumer 검색 0 hits
  (`grep -rn "ace_challenge.*_room_volume" /home/seung/mmhoa/roomestim/`):
  PATCH 범위 안 internal symbol 제거는 정상. shim 불필요. OQ로 풀지 않음
  (deferral 없음). 다음 reviewer가 "왜 shim 없냐"고 묻을 때 ADR 0030
  §Status-update-v0.15.2 §Item-H 가 명문 근거.
- **신규 ADR 없음 / 신규 D 없음 / 신규 OQ 없음** — v0.15.0 OQ-30 (per-wall α
  decomposition) 은 v0.15.2 스코프 밖, v0.15.x+ 로 유지. RoomModel evolve
  helper 는 v0.16 으로 push (v0.15.2 hygiene-only 원칙).
- **D29 lane scope 재정의 결과** — `git diff 18e776e -- roomestim_web/` 더
  이상 0 bytes 아님 (의도적; ADR 0029 §Cross-lane-geom-amendment 근거).
  D30 byte-equal 원칙은 "정책 미변경 시 core/web 양쪽 동시 touch 없음"으로
  유지 — v0.15.2는 web touch가 정책 변경 (ADR 0029 amendment) 결과이므로
  D30 위반 없음. `pyproject.toml` PATCH bump + `roomestim_web.__version__`
  MINOR bump (`0.12-web.6` → `0.12-web.7`) 양쪽 land.

## 3. 구현 (Phase 1 → Phase 4 순차)

### Phase 1 — ADR 0029 §Cross-lane-geom-amendment + ADR 0030 §Status-update-v0.15.2

- `docs/adr/0029-web-data-autofetch.md` 끝에 `## §Cross-lane-geom-amendment (v0.15.2 / 2026-05-18)` 블록 NEW (~25줄):
  - Context 1줄: v0.15.1에서 `roomestim/geom/polygon.py` shared util landed (core 내부 only). `roomestim_web/report.py` retained duplicate per cautious D29 reading.
  - Decision 본문: D29 lane separation 본래 의미는 web→core 단방향 의존 차단(core가 web을 import 하지 못함). web → core 내부의 stable public util(`roomestim.geom.*` 처럼 namespace 잠긴 leaf module) import는 lane separation을 침범하지 않음 — 오히려 duplicate 제거 측면에서 권장.
  - Scope: `roomestim_web/**`는 `roomestim.geom.polygon` (그리고 향후 추가될 `roomestim.geom.*` leaf 모듈) public 심볼 import 가능. `roomestim.model`, `roomestim.reconstruct.*` 등 기존 web→core import 경로는 영향 없음.
  - Direction (역방향 금지 강조): `roomestim/**` 가 `roomestim_web/**` import는 영구 금지. `mypy --strict roomestim/`이 web 디렉토리를 보지 않는 구성이 이 invariant를 강제.
  - Consequences: (+) `roomestim_web/report.py` duplicate 3 함수 제거 → ~24 LoC 축소. (+) 향후 geom util 진화(예: OQ-30 per-wall α decomp) 1곳만 수정. (−) `git diff` lane-isolation 무결성 검사는 "정책 변경 cycle"과 "byte-equal cycle"을 구분해야 함.
  - References: ADR 0024 §4 (D29 routing precedent), D30 (web-track versioning), v0.15.1-patch §2 결정, v0.15.2-design §2.
- `docs/adr/0030-predictor-default-switch.md` 끝에 `## §Status-update-v0.15.2 (2026-05-18)` 블록 NEW (~20줄):
  - Item G — web report geom dedup land (`roomestim_web/report.py`에서 `_polygon_area_3d` / `_shoelace_2d` / `_room_volume` 제거 → `from roomestim.geom.polygon import polygon_area_3d, room_volume` 교체). 근거: ADR 0029 §Cross-lane-geom-amendment.
  - Item H — LOW-2 clean-close. v0.15.1에서 `ace_challenge.py::_room_volume` 제거됨, deprecation shim 추가 안함. 근거: underscore prefix internal symbol + `__all__` 미노출 + grep 검색 외부 consumer 0건 + PATCH 범위 안 internal API 제거 정상. shim 강제 시 dead-code 영구화 비용 + 다음 grep에서 "왜 alias 있냐" 혼란.
  - D22 pattern 준수 (v0.10.1 / v0.15.1 precedent).
