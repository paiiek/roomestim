## 0. The defect — reproduced at HEAD `aae5514` (roomestim v0.18.2)

### 0.1 Symptom (reproduced verbatim)

A **no-op** edit produces a **non-empty** diff touching an **unrelated** speaker:

```
$ roomestim place ... → layout.yaml   (8-speaker VBAP ring; non-axis-aligned)
$ roomestim edit --speaker 0 --daz 0 ...   # zero-magnitude nudge on speaker 0
--- before
+++ after
@@ -45,7 +45,7 @@
   az_deg: -135.00000000000003
   el_deg: 0.0
   dist_m: 2.0
-  x_aim_az_deg: 44.99999999999999      ← speaker 6, NOT speaker 0
+  x_aim_az_deg: 45.0
   x_aim_el_deg: -0.0
```

The unified diff is the **headline UX** of `edit`; spurious churn on an unrelated
speaker undermines the feature. (Reproduced at plan time via the exact n8-ring
place→read→`nudge_speaker(…, daz_deg=0.0)`→write path — see §0.3.)

### 0.2 Root cause (pinned to exact lines)

The emitted numeric fields are raw `math.degrees(...)` / `math.sqrt(...)` trig
output — never normalized:

- **`roomestim/coords.py:38-48` `cartesian_to_pipeline`** produces dirty floats:
  `az = math.atan2(x, z)`, `el = math.atan2(...)`, `dist = math.sqrt(...)`. For a
  non-axis-aligned ring (e.g. az=225° → `-135.00000000000003`; az=270° →
  `-90.00000000000001`; az=315° → `-45.000000000000014`).
- **`roomestim/export/layout_yaml.py:135-149` `_placed_speaker_to_dict`** emits
  those directly: `az_deg: math.degrees(az_rad)` (line 143), `el_deg:
  math.degrees(el_rad)` (144), `dist_m: dist_m` (145), `x_aim_az_deg: aim_az_deg`
  (146), `x_aim_el_deg: aim_el_deg` (147).
- **`roomestim/export/layout_yaml.py:115-132` `_aim_az_el_deg`** computes the aim
  via a SECOND `cartesian_to_pipeline` on the negated position (line 131) — its
  own dirty floats (e.g. `44.99999999999999`, `89.99999999999999`,
  `-7.016709298534876e-15`).
- **`roomestim/export/layout_yaml.py:172-178` `placement_to_dict`** emits
  `x_wfs_f_alias_hz: float(result.wfs_f_alias_hz)` (top-level; only for WFS).

The **inconsistency** that makes a no-op edit non-empty:
`roomestim/io/placement_yaml_reader.py:28-47 `_aim_from_speaker`` reconstructs the
aim as a **unit vector** from the (dirty) `x_aim_az_deg`/`x_aim_el_deg` via
`yaml_speaker_to_cartesian(ax_deg, ael_deg, 1.0)` (line 46). The writer's second
`cartesian_to_pipeline` on that *unit* vector then lands on a **different** dirty
float than the original (e.g. `44.99999999999999` on first write vs `45.0` after
read→re-derive). Some fields re-clean, some don't — **inconsistently** — so
write≠rewrite on the non-axis case → the diff is non-empty.

## 1. The fix decision — **D56** (writer float normalization)

### 1.1 Decision matrix

| # | Option | Effort | Risk | Idempotent? | Verdict |
|---|---|---|---|---|---|
| 1 | **Normalize via `round(x, 9)` applied as the LAST step on EVERY emitted numeric degree/distance field** (place-write and edit-write share the one code path → byte-identical output → no-op edit = empty diff) | LOW (~1 helper + 5 call-site wraps in `_placed_speaker_to_dict` + 1 in `placement_to_dict`) | LOW | **YES** (idempotent fixed point; see §2.1) | **CHOSEN (D56)** |
| 2 | Quantize positions/angles to a grid earlier in `place_*` (vbap.py) | MED | MED | partial | rejected — only fixes `place`, not arbitrary reader-restored aim; the second `cartesian_to_pipeline` still re-dirties on edit-write |
| 3 | `numpy.round` / `Decimal` quantization | MED | MED | yes | rejected — heavier dep surface (numpy not a core writer dep), no benefit over stdlib `round` at this precision |
| 4 | Strip trailing-9s/0s as a YAML post-process (regex on the dumped string) | LOW | **HIGH** | yes | rejected — fragile string surgery; conflates "12 significant digits" with "dirty float", would mangle a legitimately-precise value; not the source-level normalization the user chose (approach A) |
| 5 | Custom yaml float representer | MED | MED | yes | rejected — global side-effect on all yaml dumps in-process; the writer wants a *local*, explicit, auditable transform |

### 1.2 D56 (recommended)

> **D56** — `write_layout_yaml`'s dict builder normalizes **every emitted numeric
> degree/distance field** by `round(x, 9)` applied as the **LAST step on every
> write** (approach A — normalize at the source). Fields normalized:
> **`az_deg`, `el_deg`, `dist_m`, `x_aim_az_deg`, `x_aim_el_deg`** (per-speaker,
> `_placed_speaker_to_dict`) and **`x_wfs_f_alias_hz`** (top-level,
> `placement_to_dict`). Because place-write and edit-write traverse the SAME
> `placement_to_dict` path, identical structural input → byte-identical output →
> write→read→write is an idempotent fixed point → a zero-magnitude `edit` (`--daz
> 0`) emits an **empty** diff. **Scheme**: stdlib `round(x, 9)` (NOT regex/Decimal/
> numpy/representer). **Precision**: **9 decimal places** — see §1.3 math.
> **Scope**: core (`roomestim/export/layout_yaml.py` only). **ADR ref**: ADR 0036
> §C round-trip-fidelity (enforcement *tightening*, not a policy change — D50
> Level-1 ≤1e-9 still holds, now cleaner) + §Status-update-v0.18.3; ADR 0030
> §Status-update-v0.18.3 Item Y. **D22 audit-trail-discipline** (append, no
> retroactive edits).
>
> **Rejected alternatives**: see §1.1 (options 2–5). In one line each — (2)
> place-only fix doesn't cover reader-restored aim on edit-write; (3) numpy/Decimal
> = needless dep/weight; (4) regex post-process = fragile, mangles legitimately
> precise values; (5) yaml representer = global in-process side-effect, not local/
> auditable.
>
> **Reverse-criterion (D56)**: if a real consumer needs > 9-decimal-degree
> precision in `layout.yaml` (≥ 1 report), raise N or make it a parameter and
> record in ADR 0036 §Status-update. Default stays N=9 (clean + within the 1e-9
> structural contract).

### 1.3 Precision justification — why **N = 9** (the load-bearing math)

Position error in **meters** from rounding an angle to N decimal **degrees** is
bounded by `dist · sin(Δ_az_rad)`, where the worst-case rounding error is a
half-ULP at N decimals = `0.5·10⁻ᴺ` degrees. Verified at plan time:

| N | half-ULP (deg) | pos err @ dist=2 m | vs D50 1e-9 contract |
|---|---|---|---|
| 6 | 5.0e-07 | **1.745e-08 m** | **VIOLATES** (> 1e-9) — too coarse |
| **9** | **5.0e-10** | **1.745e-11 m** | **OK** (≪ 1e-9) — **chosen** |
| 10 | 5.0e-11 | 1.745e-12 m | OK but unnecessary |
| 12 | 5.0e-13 | 1.745e-14 m | OK, approaches float64 noise floor |

- **N=9 is the sweet spot**: it is coarse enough to collapse every observed dirty
  float yet fine enough that the resulting position error (≤ `2·sin(5e-10°)` ≈
  **1.7e-11 m** for dist ≤ 2 m) is **two orders of magnitude inside** the D50
  Level-1 ≤1e-9 structural contract. N=6 would violate the contract; N≥10 buys
  nothing and re-admits dirty trailing digits.
- **Collapse verified at N=9** (plan-time): `44.99999999999999`→`45.0`,
  `-90.00000000000001`→`-90.0`, `-135.00000000000003`→`-135.0`,
  `-45.000000000000014`→`-45.0`, `89.99999999999999`→`90.0`,
  `-7.016709298534876e-15`→`-0.0`.
- **`-0.0` is preserved**: `round(-0.0, 9) == -0.0` and `yaml.safe_dump`
  serializes it as `-0.0` (verified). So existing `x_aim_el_deg: -0.0` lines stay
  byte-identical (no spurious churn), and the tiny `…e-15` aim value cleanly
  collapses to `-0.0` (a genuine, intended value change in the golden — see §3).
- **Scheme = stdlib `round(x, N)`** because it is local, explicit, dependency-free,
  and round-half-to-even is deterministic across platforms for these magnitudes.

## 2. Invariants the fix must NOT break (re-derived)

### 2.1 v0.18 byte-equal idempotency (default-aim + explicit-aim fixed points) — preserved & cleaner

`tests/test_layout_round_trip.py::test_idempotent_default_aim_rewrite_byte_equal`
and `::test_idempotent_explicit_aim_rewrite_byte_equal` (and the `roomestim
export` analogues in `tests/test_export_round_trip.py`) assert `sha(write1) ==
sha(write2)` on **axis-aligned** fixtures (az ∈ {0,90,180,270}°). **Re-derivation:**
rounding-as-last-step is **idempotent** — `round(round(x, 9), 9) == round(x, 9)`.
The first write emits `round(f(input), 9)`; the reader reconstructs a position/aim;
the second write emits `round(f(reconstructed), 9)`. Since the reader→writer cycle
on these fixtures already converges to the SAME structural input (that is *why*
they were chosen as single-iteration fixed points), and `round` is idempotent on
its own output, the second write is byte-equal to the first. The tests stay
**GREEN with no expected-value edits** because they compare write-to-self
(`sha(p1)==sha(p2)`), not write-to-a-literal. They actually become *cleaner*: the
az=270° fixture currently writes `az_deg: -90.00000000000001` on BOTH passes (dirty
but stable → byte-equal); after the fix BOTH write `az_deg: -90.0` (clean and
stable → still byte-equal). Same for `test_cli_idempotent.py` (run-twice
self-comparison, n=8 ring) and `test_export_round_trip.py::*_idempotent_second_run`.

### 2.2 D50 Level-1 ≤1e-9 structural contract (position/aim/wfs) — preserved

Per §1.3, the max position error injected by N=9 rounding is ≤ 1.7e-11 m (dist ≤
2 m), ≪ 1e-9. `dist_m` rounding error ≤ 0.5e-9 m (one half-ULP at 9 decimals) <
1e-9. Aim is a *direction* (reader unit-normalizes; D50 compares unit-normalized),
so a ≤1.7e-11 m perturbation of an angle is far inside the 1e-9 direction
tolerance. `x_wfs_f_alias_hz` at N=9 preserves ≤1e-9 (the round-trip test asserts
`approx(4000.0, abs=1e-9)`; `round(4000.0, 9) == 4000.0`).

### 2.3 `validate_placement` / engine schema validation — still passes

Schema validation (`Draft202012Validator`) checks **finiteness + type + spherical-
only structure**, not float precision. `round(finite, 9)` is finite and a `float`.
`_sweep_finite` still passes. `validate_placement` (the non-raising collector) and
the writer's in-line `validate=True` path are unaffected. (`test_edit_then_validate`
in `test_layout_round_trip.py` and the WFS-alias gates stay GREEN.)

### 2.4 Acoustic path (RT60; ADR 0009 / ADR 0030) — untouched

The change is confined to **placement serialization** (`layout_yaml.py`). No
`roomestim/reconstruct/` edit. RT60 negative control verified at plan time:
**`1.9190766987173207`** (lab_room, ISM default cascade) — must be byte-equal
post-fix. ADR 0009 ISM ≥ Eyring invariant and ADR 0030 cascade are RT60-path
concerns and are not on the placement-writer code path.

## 4. New regression gates that LOCK the fix (3 new tests)

Add to `tests/test_layout_round_trip.py` (or a focused `tests/test_writer_float_normalization.py` —
executor's choice; the round-trip module is the natural home):

- **(a) No-op edit → empty diff (the headline UX lock).** Build a **non-axis-aligned**
  layout (the n8 ring — the case that drifted), write it, then run the CLI `edit
  --speaker 0 --daz 0` (zero-magnitude nudge) and assert the unified diff between
  the input file and the written output is **EMPTY / byte-identical**. This is the
  exact dogfood reproduction turned into a permanent guard.
  ```python
  def test_noop_edit_empty_diff_non_axis_aligned(tmp_path):
      # n8 ring: az = k*45°, the case that produced -135.00000000000003 etc.
      # place → CLI edit --speaker 0 --daz 0 → assert before-bytes == after-bytes
      ...
      assert in_path.read_bytes() == out_path.read_bytes()  # empty diff
  ```
- **(b) `place` output contains NO dirty floats.** Assert every emitted numeric
  leaf equals its own `round(·, 9)` (i.e. the writer already rounded), OR
  regex-assert the dumped text contains no `…999999…` / `…000001…` /
  `…000000000…e-` patterns.
  ```python
  def test_place_output_has_no_dirty_floats(tmp_path):
      # write n8 ring; for each numeric leaf v: assert v == round(v, 9)
      ...
  ```
- **(c) Write→read→write byte-equal on a non-axis-aligned layout.** The case that
  previously needed several iterations to converge must now be a **single-iteration
  fixed point**.
  ```python
  def test_idempotent_non_axis_aligned_rewrite_byte_equal(tmp_path):
      r = _n8_ring_result()
      _write(r, p1); r2 = read_placement_yaml(p1); _write(r2, p2)
      assert _sha(p1) == _sha(p2)  # was NOT byte-equal pre-fix
  ```

These three convert the dogfood finding into executable guards so no future writer
change can silently re-introduce churn.
