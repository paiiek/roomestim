### ¬ß2.1 Scope A ‚ÄĒ CLI flags + constructive error

**File**: `roomestim/cli.py`.

1. `_add_place_parser`: add `--wfs-f-max-hz FLOAT` (default 8000.0) and
   `--wfs-spacing-m FLOAT` (default None) `add_argument` calls.
2. `_add_run_parser`: same two flags (parallel to `place`).
3. `_run_placement(...)` signature: gain
   `wfs_f_max_hz: float = 8000.0` and `wfs_spacing_m: float | None = None`
   keyword args.
4. WFS branch: derive `spacing_m` from `wfs_spacing_m` if supplied,
   else from `n_speakers` / `layout_radius` (existing formula).
5. Wrap `place_wfs(...)` in try/except `ValueError`. On
   `spacing_m > c/(2*f_max_hz)`, compute:
   - `max_safe_f_max = c / (2.0 * spacing_m)` (Hz).
   - `min_safe_n = ceil(baseline_len / (c/(2*f_max_hz))) + 1`
     (rounding up on `baseline_len * 2 * f_max_hz / c`, then +1 for
     n=intervals+1).
   Re-raise `ValueError` with a multi-clause message naming both
   remediation paths concretely. Strings used for the test assertion:
   `"max safe --wfs-f-max-hz"` and `"minimum safe --n-speakers"`.
6. `_cmd_place` and `_cmd_run`: thread `args.wfs_f_max_hz` and
   `args.wfs_spacing_m` to `_run_placement(...)` via `getattr(...)`
   defaults (defensive ‚ÄĒ `ingest` / `export` namespaces don't carry
   these flags).

## ¬ß4 Risk register

| ID | Risk | Likelihood | Mitigation |
| --- | --- | --- | --- |
| R-1 | Constructive error message math wrong (e.g. off-by-one on `min_safe_n`) | Med | Unit math: `n - 1 >= baseline_len / bound` ‚üĻ `n >= ceil(baseline_len/bound) + 1`. Verified empirically: baseline=4.0, f_max=8000, c=343 ‚Üí bound=0.0214 ‚Üí n_min = ceil(4.0/0.0214)+1 = 187+1 = 188 ‚Üí spacing=4.0/187=0.02139<0.02143 ‚úď. Test #1 asserts both clauses present (catches gross misfires); smoke shows the math live |
| R-2 | `--wfs-spacing-m` silently ignored when both flags supplied | Low | Test #4 asserts `x_wfs_f_alias_hz` reflects explicit spacing, not derived |
| R-3 | Adding flags to `place` parser breaks existing CLI tests | Low | flags are optional with non-breaking defaults; argparse silently accepts unfamiliar args only when explicitly added |
| R-4 | Breaking `_cmd_ingest` / `_cmd_export` (different argparse namespaces) | Low | Used `getattr(args, "wfs_f_max_hz", 8000.0)` defensively in `_cmd_run`/`_cmd_place` only; `_cmd_ingest` / `_cmd_export` do not call `_run_placement` |
