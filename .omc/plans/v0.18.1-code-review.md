## Stage 2 — Correctness, math, discipline

### Focus 1 — el-bound guard correctness (VERIFIED)
- **Inclusive bound:** `if not (-90.0 <= el2 <= 90.0)` accepts exactly ±90. Verified empirically:
  `del_deg=90.0` → accepted (zenith pos `(0,2,0)`), `del_deg=-90.0` → accepted (nadir pos.y=-2),
  `del_deg=90.0000001` → rejected. Boundary is correct.
- **Placement:** guard sits after `el2 = degrees(el_rad) + del_deg` (edit.py:438) and before the
  `yaml_speaker_to_cartesian` conversion (edit.py:453). Correct — rejects before the bad value can
  reach trig.
- **Interaction with `dist <= 0`:** el-guard (edit.py:445) precedes the `d2 <= 0` check
  (edit.py:451). Both are reachable and independent (different inputs). Order is immaterial to
  correctness; both raise the same `ValueError` class. No short-circuit hides either. Confirmed via
  empirical el=90.0000001 reject and existing dist-reject tests still passing.
- **No false reject / no false accept:** a legitimate nudge keeping el within [-90,90] is accepted
  (e.g. del_deg=89 → finite); a non-physical el is rejected before write. Verified.

### Focus 2 — D53 honesty: Cartesian-no-guard claim (VERIFIED TRUE)
- `cartesian_to_pipeline` (coords.py:47) computes `el = atan2(y, sqrt(x²+z²))`. The second argument
  `sqrt(...) >= 0`, so `atan2` is mathematically bounded to `[-π/2, +π/2]` = `[-90°, +90°]` for ALL
  finite inputs. Confirmed by 200k-sample random fuzz (max |el| = 89.71°) and the degenerate
  extreme y=±1e9, x=z=0 → exactly ±90.0°. A guard on the Cartesian branch would be dead code.
  D53's reasoning is honest and the comment (edit.py:443-444) is accurate.
- **Hemisphere-flip claim (comment edit.py:440-441):** verified — `el=91°` → `cos(91°)=-0.0175`
  flips x/z sign vs `el=89°` → `+0.0175`. The "silent mirror reflection, all-finite" framing in
  both the comment and RELEASE_NOTES is correct (assert_finite would not catch it).
- **Test `test_nudge_speaker_cartesian_no_el_guard` is NON-VACUOUS:** with `dy=100.0` on
  speaker 0 (orig `(0,0,2)`) the result is `(0,100,2)`, whose implied el = 88.85° (physical) — the
  test genuinely exercises the unguarded Cartesian path and asserts `p.y == 100.0` (no rejection).
  If someone wrongly added a Cartesian guard or the branch routed through the spherical path, this
  test would fail. Good regression lock.
