## 3. 구현 (Phase 1 → Phase 4 순차)

### Phase 1 — Geom shared util 추출 (LOW-1)
- `roomestim/geom/__init__.py` NEW (re-export 3 public 심볼).
- `roomestim/geom/polygon.py` NEW (~80 LoC):
  - `polygon_area_3d(polygon: object) -> float` (3D 폴리곤 면적, 기존 `_polygon_area_3d` body 이전).
  - `room_volume(room: RoomModel) -> float` (`shoelace_2d(floor) * ceiling_height`).
  - `shoelace_2d(coords: list[tuple[float, float]]) -> float`.
  - Module docstring §"Provenance": "predictor.py + ace_challenge.py 두 곳 duplicate를 v0.15.1에서 통합. D29 lane separation은 web↔core 경계이며 core 내부 추출은 OK (v0.15.1-patch §2 결정)."
- `roomestim/reconstruct/predictor.py`:
  - duplicate `_shoelace_2d` / `_polygon_area_3d` / `_room_volume` 3 함수 제거.
  - `from roomestim.geom.polygon import polygon_area_3d, room_volume, shoelace_2d` 추가.
  - callsite 4곳(line 180/183/225/228 + 266/302) 교체: `_polygon_area_3d` → `polygon_area_3d`, `_room_volume` → `room_volume`.
  - module docstring §"Layering"에서 "duplicated here" 문구 → "imported from `roomestim.geom.polygon` (v0.15.1)"로 정직성 patch.
- `roomestim/adapters/ace_challenge.py`:
  - line 919 `_room_volume` duplicate 제거 → `from roomestim.geom.polygon import room_volume` import 추가.
  - callsite 교체 (`_room_volume(room)` → `room_volume(room)`).

### Phase 2 — Per-band fallback rationale (MEDIUM-1)
- `roomestim/reconstruct/predictor.py`:
  - `_shoebox_per_band_alphas` 시그니처 변경:
    `-> tuple[tuple[float, ...], dict[int, tuple[float, ...]]]`
    `-> tuple[tuple[float, ...], dict[int, tuple[float, ...]], tuple[str, ...]]`
    (세 번째 원소 = fallback 발동 surface 이름 리스트, sorted unique).
  - 내부 `_band_alpha(surf, band_idx)` 분기에서 fallback path 진입 시 surface name(예: `wall_0`, `ceiling`, `floor`) 수집 → 외부 set으로 누적 → tuple sorted 반환.
  - `predict_rt60_default_per_band` 호출부:
    - shoebox 분기에서 `areas, per_band_alphas, fallback_surfaces = _shoebox_per_band_alphas(room)` 받음.
    - rationale base = `f"shoebox L={dims[0]:.2f} W={dims[1]:.2f} H={dims[2]:.2f}: per-band ISM (max_order={max_order})"`
    - fallback_surfaces 비어있지 않으면 `+= f"; per-band α fallback used for surfaces: [{', '.join(fallback_surfaces)}]"` append.
  - 단일 500 Hz `predict_rt60_default`는 영향 없음 (per-band 경로 미사용 → fallback 불가능; docstring에 명시).
- module docstring §"Per-band data fallbacks" patch: "v0.15.1부터 fallback 발동 시 surface 이름이 `RT60Prediction.rationale`에 누적된다" 1줄 추가.
