## 1. Mission + scope summary

v0.12-web.1 is a **parallel-track patch release** that bundles three orthogonal
themes into ONE design plan and ONE eventual commit: (A) generalise the
OBJ-only mesh path to accept `.gltf` / `.glb` / `.ply` so HF Spaces users can
drop in any common 3D scan format already supported by the vendored
`trimesh>=4.0`; (B) close the four open verifier findings (mypy availability,
extra skip-count drift, 3D viewer trace heuristic, broaden where they trace to
silent failures); (C) close the four code-reviewer HIGH/MED items (HRTF UI
attribution per CC BY 4.0, polygon-containment guard for non-rectilinear
images, broadened `_on_submit` try/except, tempdir lifecycle).
The acoustics-track core stays byte-equal; only the rename/shim of
`PolycamAdapter` (preserved as a deprecated alias) touches `roomestim/`.

## 3. Phase-by-phase implementation plan

Executor runs phases sequentially; each phase has one acceptance criterion that is independently verifiable by `pytest` before moving on. STOP rule: if any phase's acceptance fails twice in succession, halt and reopen the design plan.

### P1 — Carve `MeshAdapter` out of `PolycamAdapter` (foundation)

- **Files**: `roomestim/adapters/mesh.py` (NEW), `roomestim/adapters/polycam.py` (gut to alias), `roomestim/adapters/__init__.py` (add export).
- **Change size**: ~+195 / -90.
- **Behaviour**: `MeshAdapter.parse()` accepts the 4 mesh extensions; raises `NotImplementedError` for `.usdz` (unchanged from v0.12-web.0). `MeshAdapter` does NOT accept `.json` — the `.json` (RoomPlan sidecar) path is routed exclusively through `RoomPlanAdapter` from `roomestim_web/pipeline.py` (critic ambiguity, 2026-05-15: prior text said `MeshAdapter` accepts `.json`; corrected — the canonical route is `pipeline.py` → `RoomPlanAdapter` direct, never via `MeshAdapter`, to keep the byte-equivalence claim clean across the rename). The internal `_room_model_from_mesh()` is byte-equivalent to v0.12-web.0's `_room_model_from_obj()` except for the extension whitelist gate at the top of `parse()`. NO change to the convex-hull-of-vertex-projection geometry, NO change to the default `MaterialLabel.WALL_PAINTED` / `WOOD_FLOOR` / `CEILING_DRYWALL` triple, NO new kwargs.

- **glTF axis-convention caveat (critic What's-Missing)**: `trimesh.load(force="mesh")` flattens scene-graph transforms but does NOT correct for source-tool axis conventions (Reality Capture: Z-up; Polycam OBJ: Y-up; Apple Object Capture: variable). `MeshAdapter` uses the convex-hull-of-XY-projection (vertices[:, 0] + vertices[:, 1]) for floor polygon and (vertices[:, 2] max - min) for ceiling height. For glTF/glb files exported with a Z-up root transform, the result is geometrically valid but the "floor" and "ceiling" axes are swapped relative to the user's intent. v0.12-web.1 documents this as a known caveat in `RELEASE_NOTES_v0.12-web.1.md` § "Known gaps" and in ADR 0027 § "Consequences"; OQ-20 is broadened to cover it.
- **PolycamAdapter alias**: `polycam.py` becomes a 5-line shim — `import warnings; from roomestim.adapters.mesh import MeshAdapter; class PolycamAdapter(MeshAdapter): def parse(self, *args, **kwargs): warnings.warn("PolycamAdapter is deprecated; use MeshAdapter", DeprecationWarning, stacklevel=2); return super().parse(*args, **kwargs)`. The warning fires on call (not on import), is idempotent (Python's default `warnings.simplefilter` dedupes per location), and is silenced by `pytest`'s default warning capture without breaking existing tests.
- **Acceptance**: `pytest tests/test_adapter_polycam.py -q` → all 3 existing tests still pass byte-equal (the alias preserves the `PolycamAdapter` public surface). `pytest tests/ -m "not lab and not e2e and not web" -q` → 128 passed (no default-lane regression).
- **Test additions in this phase**: none (existing tests cover the alias path).

### P4 — Broadened `_on_submit` exception handling + tempdir lifecycle

- **Files**: `roomestim_web/app.py`, `tests/web/test_app_exception_handling.py` (NEW).
- **Change size**: ~+50 / -10.
- **Top-level guard (CRITICAL #1, critic 2026-05-15)**: in v0.12-web.0, `_on_submit` calls `result = run_pipeline(...)` on its FIRST line — OUTSIDE any try-block. If `run_pipeline()` raises (unsupported format, malformed mesh, RoomModel construction error, etc.), the user gets a Gradio crash and the per-tier guards never run. Add a top-level wrapper around the pipeline call:
  ```python
  def _on_submit(file_obj, panning, n_taps, T, gain_db, normalize):
      _LOG.info("submit start: file=%s panning=%s", file_obj, panning)
      try:
          result = run_pipeline(...)
      except Exception:
          _LOG.exception("run_pipeline failed; returning all-None tuple")
          return (None,) * 6
      # ... per-tier construction with widened excepts (below) ...
  ```
  This is the change the critic-flagged `test_app_exception_handling.py` actually exercises.
- **Per-tier exception-handling pattern**: each of the four output tiers (viewer / report / pdf / binaural) wraps its construction in
  ```python
  try:
      ...  # construct figure / report_json / pdf_str / binaural_str
  except Exception:                # was: ImportError or (ImportError, FileNotFoundError)
      _LOG.exception("tier <name> failed; returning None for this tier only")
      figure = None                # or rt60_chart, report_json, pdf_str, binaural_str

Importantly, the per-tier `None` does NOT cascade — siblings remain populated. **Note on `build_provenance_readme` + `build_result_archive`** (critic What's-Missing item): these are called UNCONDITIONALLY after the four tiers. ### P5 — Polygon-containment guard for non-rectilinear images + damping shape guard

- **Files**: `roomestim_web/binaural.py`, `tests/web/test_binaural_polygon_containment.py` (NEW), `tests/web/test_binaural_damping_shape.py` (NEW).
- **Change size**: ~+40 / -8.
- **Polygon guard (HIGH-2 option (a) chosen)**: new helper at `roomestim_web/binaural.py:~70`:
  ```python
  def _image_inside_floor(image_xz: tuple[float, float], floor_polygon: Sequence["Point2"]) -> bool:
      """True if image-source xz projection is inside the floor polygon.

      Shoebox path: this is always true by construction (AABB ⊆ polygon).
      Extrusion path: pyroomacoustics generates mirror images that may fall
      outside concave bites of the polygon — drop those with a warning.
      """
      from shapely.geometry import Point, Polygon
      poly = Polygon([(float(p.x), float(p.z)) for p in floor_polygon])
      return bool(poly.contains(Point(float(image_xz[0]), float(image_xz[1]))))
  ```

## 5. Decision-log entries to append (D32 + D33)

### D32 — Tempdir lifecycle: bounded deque + atexit reaper, NOT per-call cleanup

> **Append format note (critic 2026-05-15)**: when these two entries land in `.omc/plans/decisions.md`, the leading `###` H3 marker is promoted to `##` H2 to match the file's existing D1–D31 H2 convention. The H3 in this plan is only for the in-plan structural nesting under §5.

**Date**: 2026-05-15b (v0.12-web.1).

**Context**: `_on_submit` in v0.12-web.0 used `tempfile.mkdtemp(prefix="roomestim_")` and never cleaned up. HF Spaces deploys the demo on an ephemeral container with bounded disk; abandoned tempdirs accumulate and eventually OOM-the-disk the container. But Gradio's `gr.File` output requires the file to remain on disk after `_on_submit` returns (the user downloads it lazily on tab navigation).

**Decision**: Hold the last 8 `TemporaryDirectory` instances in a module-level `deque(maxlen=8)`. Deque eviction triggers `__exit__` on the oldest entry, which `shutil.rmtree`s its directory. An `atexit`-registered reaper walks `tempfile.gettempdir()` at process shutdown for any `roomestim_*` directories older than 4 h (`mtime` heuristic) and removes them. The 8-entry cap covers the typical HF Spaces user session; the 4 h `atexit` window catches abandoned containers without breaking active downloads.

**Alternatives considered**:
- **(a) `tempfile.TemporaryDirectory()` with `gr.State`-tracked cleanup**: Gradio's `gr.State` does not survive page reload, and the cleanup hook fires on user navigation, which can happen before the download completes. Rejected — race condition.
- **(b) `atexit`-only reaper without the deque cap**: process never exits in HF Spaces (long-running gunicorn worker), so atexit never fires until container cycle. Without the deque cap, every submit leaks until cycle. Rejected — leak window too large.
- **(c) Document as known and ignore**: rejected per code-reviewer MED-2 — HF Spaces disk OOM is a real failure mode, not a theoretical one.

**Reverse-criterion**: If OQ-18 (HF Spaces cold-start) measurement reveals containers cycle in < 1 h, tighten to 30-min `mtime` window at v0.12-web.2 per OQ-22 resolution path. If the 8-entry deque cap proves too low for power users (i.e. anyone who submits > 8 times in one session), raise to 16 or switch to a 2 h TTL eviction.

**Cross-ref**: §3-P4; OQ-22; `roomestim_web/app.py:_TEMP_REAPER` + `_reap_stale_tempdirs()`.

### D33 — MeshAdapter rename (NOT shim): single canonical class; PolycamAdapter retained as deprecated subclass alias

**Date**: 2026-05-15b (v0.12-web.1).

**Context**: v0.12-web.1 generalises the OBJ-only path to four mesh formats (`.obj`, `.gltf`, `.glb`, `.ply`). Two surfacing options were available: (i) rename `PolycamAdapter` → `MeshAdapter` and re-export the old name as a deprecated alias; (ii) keep `PolycamAdapter` as-is and add a new sibling `MeshAdapter` class that delegates.

**Decision**: Option (i) — rename. `MeshAdapter` lives in `roomestim/adapters/mesh.py` and carries the full mesh-parsing logic. `PolycamAdapter` becomes a 5-line subclass shim in `roomestim/adapters/polycam.py` that emits one `DeprecationWarning` on first `.parse()` call per location. The rename is the cleaner option because: (a) the v0.1+ class name no longer matches its actual scope (it parses much more than Polycam exports); (b) the convex-hull-of-projection geometry is mesh-format-agnostic, so a subclass adds no behavioural value; (c) external callers (incl. `pipeline.py`, `test_adapter_polycam.py`, `test_binaural_renderer.py`, `test_3d_viewer.py`, `test_pipeline_integration.py`) can migrate to `MeshAdapter` at their own pace via the alias; (d) the alias-as-subclass pattern is `isinstance(PolycamAdapter(), MeshAdapter)` → True, so any duck-typing call sites continue to work byte-equal.

**Alternatives considered**:
- **(ii) New sibling `MeshAdapter` delegating to `PolycamAdapter`**: rejected because it freezes the misleading `PolycamAdapter` name as the canonical implementation, perpetuating the v0.1-era misnomer; also doubles the maintenance surface (every behaviour change requires touching two classes).
- **(iii) Add a `format=` kwarg to `PolycamAdapter.parse()` without rename**: rejected because the suffix dispatch already exists at the top of `parse()`; a kwarg would be redundant noise.

**Reverse-criterion**: If a v0.13+ user reports the `DeprecationWarning` is too noisy in HF Spaces logs, downgrade to `PendingDeprecationWarning` at v0.13-web.0. If a v0.14 release decides to drop `PolycamAdapter` entirely (full removal of the alias), that lands under a successor D-decision and a `RELEASE_NOTES_v0.14-web.0.md` "Breaking changes" callout.

**Cross-ref**: §2.2 (`roomestim/adapters/polycam.py` shim); §3-P1; ADR 0027; v0.12-web.0 ADR 0024 (parallel-track package boundary preserved); D6 (convex-hull-of-projection caveat unchanged).

## 6. ADR sketches (1 paragraph each)

### ADR 0027 — Mesh-format generalisation via single MeshAdapter

**Status**: Accepted (v0.12-web.1).

**Decision**: Generalise the OBJ-only `PolycamAdapter` into a single mesh-format-agnostic `MeshAdapter` (new module `roomestim/adapters/mesh.py`) accepting `.obj`, `.gltf`, `.glb`, `.ply` via `trimesh.load(force="mesh")`. The convex-hull-of-vertex-projection geometry, the default `MaterialLabel` triple (`WALL_PAINTED` + `WOOD_FLOOR` + `CEILING_DRYWALL`), and the `RoomModel.schema_version="0.1-draft"` are all preserved byte-equal across the four mesh formats. `PolycamAdapter` is retained as a `DeprecationWarning`-emitting subclass alias (D33). `.usdz` continues to raise `NotImplementedError` (no `[usd]` extra in default CI), and `.json` (RoomPlan sidecar) continues to delegate to `RoomPlanAdapter`. **Drivers**: (i) HF Spaces users export room scans from a mix of Polycam (`.usdz`/`.obj`), Reality Capture (`.gltf`), Apple Object Capture (`.ply`), and Sketchfab (`.glb`) — accepting all four with no new core dependency closes the on-ramp for the v0.12-web.0 release. (ii) `trimesh>=4.0` is already pinned in `pyproject.toml` core dependencies and natively loads all four. (iii) Renaming (rather than shimming) avoids freezing the v0.1-era misnomer (D33). **Consequences**: (a) `PolycamAdapter` deprecation surfaces in pytest only when a caller invokes `.parse()` (one `DeprecationWarning` per location); existing tests pass byte-equal because pytest's default capture mode dedupes and does not fail on warnings. (b) glTF scene-graph semantic material labels are NOT extracted; `trimesh.load(force="mesh")` flattens to a single mesh and the convex-hull geometry uses only vertex coordinates — same caveat as `.obj`. (c) Alpha-shape reconstruction (D6) deferred to v0.3 stays in scope; v0.12-web.1 inherits the convex-hull caveat byte-equal across all 4 mesh formats. **Follow-ups**: OQ-20 (`.glb` byte-equal reproducibility), OQ-21 (PLY no-faces degenerate case).

(No second ADR is needed for v0.12-web.1 — D32 (tempdir lifecycle) is an implementation choice within ADR 0024's parallel-track package boundary; the cross-reviewer findings closed in §3-P4/P5/P6 are tactical fixes within ADR 0024/0025/0026 scope. ADR 0027 is the only structural decision warranting an ADR file.)

## 9. Rollback plan

If v0.12-web.1 breaks the HF Spaces deploy or fails the verifier gate post-ship, the rollback path has three tiers ordered by least → most invasive:

### Tier 1 — Per-phase revert (single-phase failure)

Per-phase revert is **NOT universally clean** (critic MAJOR #3, 2026-05-15). The phase-dependency graph is:

| Phase | Depends on | Cleanly revertable in isolation? |
|---|---|---|
| P1 (MeshAdapter carve-out) | — | YES (but reverting P1 ⇒ MUST also revert P2 + P3) |
| P2 (fixtures + parametric test) | P1 | NO standalone — `test_adapter_mesh.py` references `MeshAdapter` |
| P3 (pipeline + UI + attribution) | P1 (uses `MeshAdapter`) | NO standalone for pipeline.py route table; attribution markdown block IS standalone-revertable |
| P4 (`_on_submit` + tempdir reaper) | — | YES — orthogonal to P1/P2/P3 |
| P5 (polygon + damping guards) | — | YES — orthogonal to all others |
| P6 (mypy skipif + version bump + verifier closures) | — | YES — orthogonal |
| P7 (docs + ADR + release notes) | every phase (docs describe them) | YES — partial revert of release-notes paragraphs corresponding to the reverted code phase |

**Cleanly revertable alone**: P4, P5, P6, P7-partial. If only one of these fails, the executor reverts THAT phase's files and re-commits the remaining work as `v0.12-web.1-partial`.

**Bundle-revert required**: P1 ⇒ also revert P2 + P3 + the `MeshAdapter` references in P7's ADR/release-notes. If P1 fails, the bundle reverts together (i.e. mesh-format generalisation as a whole reverts, leaving the reviewer/verifier closures from P4/P5/P6 shipped).

### Tier 2 — Whole-patch revert (multi-phase failure or HF Spaces deploy break)

`git revert <v0.12-web.1-sha>` cleanly returns the repo to v0.12-web.0 state. Acoustics-track byte-equal stays untouched (only the `roomestim/adapters/polycam.py` shim + the new `mesh.py` reverts; the `RoomModel` schema, MaterialLabel enum, and all α values are completely outside the revert surface). The deprecated `PolycamAdapter` alias unwinds back to the v0.12-web.0 canonical class; `MeshAdapter` disappears from the public API. The 4 new mesh-format fixtures (`lab_room.gltf/.glb/.ply`) are also reverted but cause no test breakage because the v0.12-web.0 test suite never referenced them. ADR 0027 + D32 + D33 + OQ-20/21/22 are reverted in lockstep.

### Tier 3 — Forward-fix (data-corruption or behaviour regression discovered post-ship)

If the rollback would itself break something (e.g. a downstream consumer has already pinned v0.12-web.1 in their `requirements.txt`), ship v0.12-web.2 as a forward-fix that explicitly disables the offending phase (e.g. force the pipeline to route only `.obj` to `MeshAdapter` and reject the other three formats with a `NotImplementedError` until the fix lands). A v0.12-web.2 hotpatch leaves the v0.12-web.1 tag intact but supersedes it in `__version__`. Record the hotpatch in a `RELEASE_NOTES_v0.12-web.2.md` "Hotpatch from v0.12-web.1" callout and append a `## Status update YYYY-MM-DD` block to `.omc/plans/v0.12-web.1-design.md` documenting which phase tier(s) were forward-fixed.

### Rollback acceptance gate

Whichever tier fires, the post-rollback state MUST pass:
- `pytest -m "not lab and not e2e and not web" -q` → ≥ 128 passed (the v0.12-web.0 baseline).
- `pytest -m web -q` → ≥ 21 passed + 1 skipped (the v0.12-web.0 baseline).
- `python -c "import roomestim_web; print(roomestim_web.__version__)"` → returns either `0.12-web.0` (Tier 2) or `0.12-web.2` (Tier 3) — NOT a dangling `0.12-web.1` string.
