## 3. Phase별 구현

### Phase 1 — WFS UX (A4)

- `roomestim_web/app.py` (+30/-5), `pipeline.py` (+5/-2), `tests/web/test_wfs_ux.py` (NEW).
- (a) `gr.Slider(500, 8000, value=1500, visible=False)`; (b) `algorithm.change`로 WFS 시 visible; (c) `_on_submit`/`run_pipeline` signature에 `wfs_f_max_hz` 추가 → `dispatch.run_placement(wfs_f_max_hz=...)` 전달; (d) Exception 시 `report_json={"error": str(exc), "algorithm": algorithm}` surface, 나머지 5 None.

### Phase 2 — fetch 스크립트 (B1)

- `scripts/fetch_web_data.py` 재작성 (+180/-65).
- `argparse`: `--dry-run` (기본), `--download`, `--data-dir`. HUTUBS = urlretrieve + zipfile.extract + SHA-256; KEMAR = SOFAconventions GitHub mirror; LibriVox = urlretrieve + `ffmpeg -ss 0 -t 30 -ar 48000 -ac 1` subprocess. Idempotent (SHA 일치 시 skip).
- URL slots (executor 채움): `HUTUBS_ZIP_URL`, `KEMAR_SOFA_URL`, `LIBRIVOX_MP3_URL`.

### Phase 3 — app.py 통합 (B2+B3)

- `roomestim_web/app.py` (+45/-5).
- (a) `_ensure_web_data()` — 파일 체크 + `threading.Thread(daemon=True)` non-blocking fetch; env opt-out. (b) `build_demo()` 직전 1회 호출. (c) `_on_submit` 바이노럴 분기 데이터 미존재 시 `report_json["binaural_status"]="데이터 미준비 — python scripts/fetch_web_data.py --download 실행"` + 탭 `gr.Markdown` placeholder.

### Phase 4 — 테스트 + 문서

- NEW: `test_wfs_ux.py` (2), `test_fetch_web_data.py` (3, `monkeypatch` mock urlretrieve), `test_binaural_fallback.py` (1); `docs/adr/0029-web-data-autofetch.md`; `.omc/plans/decisions.md` D36; `open-questions.md` OQ-26; `RELEASE_NOTES_v0.12-web.4.md`.
