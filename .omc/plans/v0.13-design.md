### §0.1 Why-this-scope-not-alternatives

This subsection records the explicit rejection rationales for the four
most-likely-alternative scope shapes that the v0.13 planner round
considered and rejected.

**(a) "Include OQ-15 ISM bundle at v0.13"** — REJECTED.
The OQ-15 ISM bundle = new library code `roomestim/predict/image_source.py`
(~300-500 LOC) + ADR 0022 NEW + per-room ISM-vs-Sabine ratio
characterising tests + analytic shoebox validation + perf doc. That is
structurally a DELIBERATE bundle on its own terms (≈ v0.11/v0.12-equivalent
substantive scope). Bundling it with Items A/B/C/D at v0.13 replays the
v0.11 §0.1 enum-scope-lock anti-pattern that D23 §Why-this-scope (b)
explicitly rejected ("Ship all 4 + FIBERGLASS_CEILING + TILE_FLOOR" —
same minimum-leverage failure mode). D26 (predictor cadence) and D27
(citation cadence) are INDEPENDENT cadences — coupling them at v0.13
is coincidence, not synergy. v0.14 = ISM bundle + D27 hard-wall closure
(both naturally land together because they are both v0.X+3 in their
respective cadences). Main-agent tiebreaker analysis (Critic-lean
ruling) confirmed.

**(b) "Full Vorländer book purchase or ILL request"** — REJECTED at v0.13
design time per explicit user lock "도서관 구매 안할경우의 베스트로 진행"
(under no-library-purchase constraint, ship the best path). v0.13 ships
path α (SECOND re-deferral) per §0.0 Item A row. The "ILL / library
acquisition" path is itself a v0.14 hard-wall option (per ADR 0019 §References
+ D27 reverse-criterion) — if it materialises by v0.14, the hard wall
closes via path (i) of the D27 schedule. v0.13 does NOT pre-commit to
the v0.14 closure path; it preserves all three v0.14 options (verbatim
acquired / PRIMARY source switch to Bies & Hansen / PRIMARY source
switch to NRC).

**(c) "Wider lint scope than docs/perf + docs/architecture (i.e., go to
`.omc/plans/*.md` too)"** — REJECTED.
The `.omc/plans/*.md` family contains intentional citation-of-pattern
text (e.g., `.omc/plans/v0.11-design.md:75` literally cites `"we ship\|ship in v0\.[0-9]"`
as a grep pattern). Adding planner internal docs to lint scope would
generate a cascade of false positives that would either (a) require >
10 `# noqa: lint-tense — pattern citation` annotations across the plan
files (eroding the lint's signal-to-noise ratio); or (b) require a more
sophisticated exclusion mechanism (e.g., backtick-fenced code-block
exclusion) that is itself a v0.14+ structural change to ADR 0020 — out
of v0.13 SHORT envelope. v0.13 stops at `docs/*.md` + `.omc/research/*.md`
which together represent the natural "narrative docs" layer; planner
