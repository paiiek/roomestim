## Stage 1 — Spec compliance (PASS)

The patch solves exactly the stated defect: no-op `edit --daz 0` emitted spurious
cross-speaker diff noise from raw `math.degrees`/trig dirty floats. D56 normalizes
all emitted numeric fields to 9 dp at emit so place-write and edit-write share one
code path → byte-identical → no-op edit = empty diff. Nothing missing, nothing extra.

## Stage 2 — Focus-area findings (all empirically verified)

### 1. Fix correctness & completeness — PASS
- `_round9` applied to ALL 6 emitted numeric fields: per-speaker `az_deg`, `el_deg`,
  `dist_m`, `x_aim_az_deg`, `x_aim_el_deg` (`_placed_speaker_to_dict`, layout_yaml.py:166-170)
  + top-level `x_wfs_f_alias_hz` (`placement_to_dict`, :201). No numeric leaf missed
  (`id`/`channel` are ints; `version`/`name`/`regularity_hint` are strings).
- Rounding is the LAST op before serialization (wraps the value placed into the dict;
  `yaml.safe_dump` consumes the dict unchanged). Idempotent regardless of upstream trig noise.
- `-0.0` preserved: `round(-0.0, 9)` → `-0.0` (signbit verified negative). Golden keeps
  `x_aim_el_deg: -0.0` lines intact.
- D50 Level-1 ≤1e-9 contract NOT violated: meters err = `2·sin(radians(0.5e-9°))`
  = **1.7453e-11 m** at dist=2m — verified, two orders of magnitude inside 1e-9.
- Idempotency `round(round(x,9),9)==round(x,9)`: verified over 100k random samples, holds.
