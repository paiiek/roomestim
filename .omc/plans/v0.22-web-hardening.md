## Context

The tool is a publicly-deployable Gradio app (`app.py` repo-root shim → `roomestim_web.app`,
HF Spaces). The v0.20.0 multi-perspective security audit found **NO CRITICAL/RCE**
(YAML `safe_load` uniform, no eval/exec/pickle/shell). OQ-45 collects five code-level

## Work Objectives

1. Bound untrusted-mesh parsing (DoS guard) in `MeshAdapter` (protects CLI **and** web)
   plus a Gradio `max_file_size` cap at the upload boundary.

   + README `sdk_version` reconcile); defer CI `pip-audit` + lockfile to a residual OQ.

## Risk register

- **R-1 (flagged, dep floors)**: a gradio floor bump that implies a major (4→6) could in a
  fresh install pull an API-incompatible gradio. **Mitigation**: floors are declaration-only
  and the installed env (6.14.0) already satisfies a higher floor; pick a floor ≤ installed
  version (e.g. `gradio>=4.44`) so the contract widens-forward without forcing a downgrade or
  an untested major. If unsure on any package, defer to OQ-46. This is the only genuinely
  risky item; everything else is mechanical and unit-tested.
- **R-2 (low)**: gradio `max_file_size` kwarg location varies by version. **Mitigation**:
  prefer `launch(max_file_size=...)` (stable); verify against installed 6.14.0 at executor time.
- **R-3 (low)**: per-PID reaper means a crashed process's dirs are no longer reaped by a fresh
  process. **Accepted**: HF Spaces containers cycle ≤ 24 h (OQ-22 context) and the OS clears
  the temp root on container recycle; cross-session-deletion safety outweighs orphan risk.

## ADR (for the recommended ADR 0038 — input resource bounds)

- **Decision**: Bound untrusted mesh input (file bytes + vertex count) inside `MeshAdapter`
  via env-overridable module constants; raise a clear `ValueError`. Apply a matching Gradio
  `max_file_size` at the web upload boundary.
- **Drivers**: publicly-deployable Gradio app reaches `trimesh.load(force="mesh")` unbounded;
  trivial DoS on a shared Space; defense-in-depth (adapter bound protects CLI too, web cap
  rejects early before bytes hit disk).
- **Alternatives considered**: (a) web-only cap (rejected — leaves CLI/library unguarded and
  the bound untestable without a browser); (b) hard-coded limits (rejected — env override
  needed for legitimate large-scan operators).
- **Why chosen**: adapter-level bound is the single chokepoint reachable from every entry
  path and is unit-testable by monkeypatching the constant low; env override preserves
  flexibility; module constants keep the policy discoverable.
- **Consequences**: `MeshAdapter.parse` gains a new public error path (SemVer-MINOR);
  future adapters should adopt the same bound pattern.
- **Follow-ups**: OQ-46 (CI `pip-audit` + lockfile).
