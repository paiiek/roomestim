# RELEASE NOTES — roomestim v0.5.0

v0.5.0 ships partial-A (ACE geometry dims verified against arXiv:1606.03365 Table 1) +
B (MISC_SOFT enum extension). The ACE corpus byte-audit resolves the dimensional half
of v0.4's DEFERRED Finding 1, patching one genuine numerical error (Office_2). Material
assignments, the Lecture_2 ceiling hypothesis, and the coefficient revision all remain
DEFERRED pending TASLP access.

A12 byte-equality of every v0.4.0 default-lane test is preserved (80 → 84;
+4 MISC_SOFT unit tests; nothing existing is modified).

---

## Highlights

- **ACE adapter Office_2 dimensional fix**: `W` 3.50 → 3.22 m, `H` 3.00 → 2.94 m
  (V_m³ 53.55 → 48.28; ~10% shrink). Source: arXiv:1606.03365 Table 1 (open access).
  Gated audit test + committed CSV fixture ensure the fix is reproducible.
- **MISC_SOFT enum slot**: `MaterialLabel.MISC_SOFT = "misc_soft"` added to the
  closed enum. `MaterialAbsorption[MISC_SOFT] = 0.40`;
  `MaterialAbsorptionBands[MISC_SOFT] = (0.20, 0.30, 0.40, 0.50, 0.60, 0.65)`.
  Schema slot reserved for adapter-emitted furnishings/occupants absorption budget.
  No adapter emits MISC_SOFT at v0.5.0; wiring follows consumer demand.

## Perf doc regeneration

The gated E2E test at `tests/test_e2e_ace_challenge_rt60.py` writes its
report to `docs/perf_verification_e2e_2026-05-07.md` (the pinned path was
bumped from `2026-05-06.md` to `2026-05-07.md` for v0.5 so the v0.4 doc is
preserved for historical diff). Regenerate via:

```bash
ROOMESTIM_E2E_DATASET_DIR=/tmp/ace_corpus \
    python3 \
    -m pytest tests/test_e2e_ace_challenge_rt60.py::test_e2e_rt60_characterisation \
    -m e2e -s
```

Office_2 numbers shift due to the V_m³ correction (53.55 → 48.28 m³):
- 500 Hz: Sabine 0.887 → 0.862 s, Eyring 0.837 → 0.814 s.
- err Sabine +0.435 → +0.410 s, err Eyring +0.385 → +0.362 s.

Other rooms unchanged. The Eyring monotonicity invariant (`eyring ≤ sabine + 1e-9`)
is asserted at runtime in the E2E test and continues to hold across all 7 rooms
and 6 octave bands.
